# 睡眠窗口测试用例

> 覆盖推荐算法、圆盘交互、高能预览、提醒设置、接口降级和边界情况。
> 每条用例包含前置条件、操作步骤和预期结果。
> 测试类型：`自动化` = 单元测试/集成测试可直接覆盖；`手动` = 需人工验证。

---

## 自动化测试

### A. 推荐算法（单元测试）

> 推荐逻辑为纯函数，输入历史数据 + 当前时间 → 输出推荐时间。Mock 时钟和数据即可。

#### A01 正常推荐 · 同星期几数据充足

**输入**：今天周三，过去 3 个周三的起床时间 [06:15, 06:30, 06:45]，过去 7 天睡眠数据完整。

**断言**：
- `regular_wake_time == "06:30"`
- `recommended_bedtime == optimal_bedtime - 30min`
- `basis.data_days == 7`

#### A02 降级推荐 · 同星期几数据不足

**输入**：今天周六，过去 3 周仅 1 个周六有数据（07:30），过去 7 天整体起床均值 06:40。

**断言**：
- `regular_wake_time == "06:40"`（降级为 7 天均值，而非 07:30）

#### A03 降级推荐 · 数据不足 7 天

**输入**：仅有 3 天睡眠数据。

**断言**：
- `basis.data_days == 3`
- 推荐正常返回（不报错）

#### A04 超过推荐入睡时间后打开

**输入**：`recommended_bedtime = "23:00"`，`current_time = "23:20"`。

**断言**：
- 展示用的推荐入睡时间 == `"23:30"`（当前时间后最近 15 分钟刻度）
- 起床时间不变

#### A05 超过推荐入睡时间后打开 · 整点

**输入**：`recommended_bedtime = "23:00"`，`current_time = "23:00"`。

**断言**：
- 展示用的推荐入睡时间 == `"23:00"`（无调整）

#### A06 凌晨打开 · 5 点前

**输入**：`current_time = "02:15"`，`recommended_bedtime = "23:00"`。

**断言**：
- 仍视为"今晚"
- 展示用的推荐入睡时间 == `"02:30"`

#### A07 凌晨打开 · 5 点后

**输入**：`current_time = "05:15"`。

**断言**：
- 视为新的一天
- 展示用的推荐入睡时间 == 正常计算值（不做时间调整）

### B. 高能预览文案（单元测试）

> 时长格式化和文案拼接为纯函数，输入分钟数 → 输出字符串。

#### B01 推荐态展示

**输入**：`total_min = 450`，`delta_vs_recommended = 0`，`data_days = 7`，`recommended_bedtime = "23:00"`。

**断言**：
- 时长文案 == `"7 小时 30 分"`
- 说明文案包含 `"基于你过去 7 天的睡眠数据"`
- 说明文案包含 `"23:00 入睡可实现明日高能最大化"`

#### B02 调整态 · 高能减少

**输入**：`total_min = 312`，`delta_vs_recommended = -138`，推荐睡眠 450min，当前睡眠 360min。

**断言**：
- 时长文案 == `"5 小时 12 分"`
- 说明文案包含 `"睡眠比推荐减少 1 小时 30 分"`
- 说明文案包含 `"明日高能减少 2 小时 18 分"`

#### B03 调整态 · 高能增加

**输入**：`total_min = 480`，`delta_vs_recommended = 30`，推荐睡眠 450min，当前睡眠 510min。

**断言**：
- 时长文案 == `"8 小时 0 分"`
- 说明文案包含 `"睡眠比推荐增加 1 小时 0 分"`
- 说明文案包含 `"明日高能增加 30 分钟"`

#### B04 调整态 · 高能无变化

**输入**：`total_min = 450`，`delta_vs_recommended = 0`，推荐睡眠 450min，当前睡眠 465min。

**断言**：
- 说明文案包含 `"明日高能无明显变化"`

#### B05 格式 · 不足 1 小时

**输入**：`total_min = 45`。

**断言**：
- 时长文案 == `"45 分钟"`（省略小时）

#### B06 格式 · 整小时

**输入**：`total_min = 360`。

**断言**：
- 时长文案 == `"6 小时 0 分"`

#### B07 格式 · 差值不足 1 小时

**输入**：`delta_vs_recommended = -45`，睡眠差值 45min。

**断言**：
- 差值文案中出现 `"45 分钟"`（省略小时）

### C. 接口与降级（集成测试）

> Mock API 返回值，验证客户端状态机切换。

#### C01 init 成功

**Mock**：init 返回正常数据。

**断言**：
- skeleton 消失
- 圆盘位置 == 推荐值
- 高能预览 == 推荐态文案
- 提醒开关可交互

#### C02 init 失败 · 无睡眠数据

**Mock**：init 返回 `NO_SLEEP_DATA`。

**断言**：
- 首页「入睡窗口」卡片不展示

#### C03 init 失败 · 服务端异常

**Mock**：init 返回 `INTERNAL_ERROR`。

**断言**：
- 页面展示"加载失败"
- 存在重试按钮
- 提醒开关置灰不可交互
- 点击重试 → 重新发起 init 请求

#### C04 predict 成功

**Mock**：predict 返回 `total_min: 312, delta_vs_recommended: -138`。

**断言**：
- 高能预览更新为新数据
- 文案切换为调整态

#### C05 predict 超时

**Mock**：predict 3 秒不返回。

**断言**：
- 高能时长显示 `"--"`
- 文案显示 `"预测暂不可用，请稍后再试"`
- 提醒开关仍可操作

#### C06 predict 失败

**Mock**：predict 返回错误。

**断言**：同 C05。

#### C07 网络断开

**Mock**：网络不可用。

**断言**：
- 拖动本地交互正常（圆心时长 + 顶部时间更新）
- 松手后显示 `"网络不可用，无法预测"`

### D. 提醒逻辑（集成测试）

> Mock NotificationCenter 和系统闹钟 API，验证注册/取消/文案选择逻辑。

#### D01 开启入睡提醒 · 推荐时间

**前置条件**：未调整时间，`recommended_bedtime = "23:00"`，`optimal_bedtime = "23:30"`。

**操作**：打开入睡提醒开关。

**断言**：
- NotificationCenter 注册调用参数：触发时间 == 23:00，重复 == daily
- 推送 body 包含 `"23:00–23:30"` 且包含 `"最佳入睡窗口"`

#### D02 开启入睡提醒 · 自定义时间

**前置条件**：入睡时间已拖到 00:00。

**操作**：打开入睡提醒开关。

**断言**：
- NotificationCenter 注册调用参数：触发时间 == 00:00
- 推送 body 包含 `"00:00"`
- 推送 body **不包含** `"最佳入睡窗口"`

#### D03 关闭提醒

**前置条件**：入睡提醒已开启。

**操作**：关闭入睡提醒开关。

**断言**：
- NotificationCenter removePendingNotification 被调用

#### D04 时间调整联动

**前置条件**：入睡提醒已开启（推荐态 23:00）。

**操作**：拖动入睡时间到 00:00。

**断言**：
- 旧推送被取消
- 新推送注册：触发时间 == 00:00，文案为自定义态

#### D05 调整回推荐时间后联动

**前置条件**：接 D04，入睡提醒已开启，自定义态。

**操作**：拖回 23:00–06:30。

**断言**：
- 推送注册恢复：触发时间 == 23:00，文案包含 `"最佳入睡窗口"`

#### D06 状态持久化

**操作**：设置入睡提醒开启 + 入睡时间 00:00 → 退出页面 → 重新进入。

**断言**：
- 入睡提醒开关 == 开启
- 持久化存储中 selected_bedtime == "00:00"

#### D07 页面刷新联动

**前置条件**：上次入睡提醒开启（推荐态 23:00），本次 init 返回 `recommended_bedtime = "22:45"`。

**断言**：
- 推送触发时间更新为 22:45
- 推送文案中包含新的入睡窗口时段

## 手动测试

### M01 圆盘拖动 · 就寝点推迟

**操作**：将就寝点从 23:00 拖到 00:30。

**验证**：
- 起床时间不变（06:30）
- 圆心时长更新为 6h 00min
- 顶部就寝时间更新为 00:30
- 松手后高能预览更新为调整态

### M02 圆盘拖动 · 就寝点提前

**操作**：将就寝点从 23:00 拖到 21:30。

**验证**：
- 圆心时长更新为 9h 00min
- 松手后高能预览更新

### M03 圆盘拖动 · 起床点

**操作**：将起床点从 06:30 拖到 05:00。

**验证**：
- 就寝时间不变（23:00）
- 圆心时长更新为 6h 00min

### M04 圆盘拖动 · 整体弧线平移

**操作**：拖动睡眠弧线中间部分向后平移。

**验证**：
- 睡眠时长不变（如始终 7h 30min）
- 就寝和起床时间同步移动
- 松手后高能预览更新

### M05 拖动触达最小时长 2h

**操作**：拖动就寝点使时长接近 2h，继续拖。

**验证**：
- 到 2h 后拖动点停止
- 不可继续缩短

### M06 拖动触达最大时长 10h

**操作**：拖动就寝点使时长接近 10h，继续拖。

**验证**：
- 到 10h 后拖动点停止
- 不可继续拉长

### M07 15 分钟吸附

**操作**：拖动就寝点到 23:07 附近，松手。

**验证**：
- 拖动点吸附到 23:00 或 23:15
- 顶部时间显示吸附后的值

### M08 快速连续拖动

**操作**：快速拖动松手 → 立即再拖松手 → 再拖松手。

**验证**：
- 高能预览只显示最终位置的结果
- 不出现中间状态闪烁

### M09 拖回推荐位置

**操作**：先拖偏，再拖回推荐的 23:00–06:30。

**验证**：
- 高能预览恢复为推荐态文案

### M10 通知权限弹窗

**前置条件**：系统通知权限未授权。

**操作**：打开入睡提醒开关。

**验证**：
- 弹出系统授权弹窗
- 拒绝 → 开关自动回关
- 允许 → 正常注册推送

### M11 起床闹钟

**操作**：打开起床闹钟开关。

**验证**：
- 系统闹钟 App 中出现对应时间的闹钟

### M12 状态持久化 · 杀进程恢复

**操作**：设置入睡提醒开启、入睡时间 00:00 → 杀进程 → 重新打开。

**验证**：
- 入睡提醒开关恢复为开启
- 入睡时间恢复为 00:00（或更新为新推荐值）

### M13 卡片点击进入

**操作**：点击首页「入睡窗口」卡片。

**验证**：
- 跳转到睡眠窗口页面
- skeleton → 加载完成后展示推荐态

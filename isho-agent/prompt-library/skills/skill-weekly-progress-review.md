# Skill: 周度进展回顾

> 触发条件：每周固定时间（如周日晚间或周一早晨）主动触发，或用户主动请求（"这周情况怎么样""帮我总结一下"）

---

## 完整流程

### Step 1 — 拉取 14 天数据 + 策略

并行调用三组数据：

```
show_status("正在整理这两周的数据…")

get_strategy(aspects: ["trends", "active", "history"])
get_health_data(metrics: ["sleep_stages", "hrv_sdnn", "sleep_debt", "sleep_consistency"], range: "14d")
get_health_data(metrics: ["high_energy_hours_total", "steps"], range: "14d")
```

拉 14 天而非 7 天，是为了做**本周 vs 上周**对比。

---

### Step 2 — 本周 vs 上周 Markdown 表格

用结构化表格呈现关键指标的周对比：

```markdown
| 指标 | 上周均值 | 本周均值 | 变化 |
|------|----------|----------|------|
| 平均入睡时间 | 00:35 | 00:12 | 早了 23min ✦ |
| 平均睡眠时长 | 6.3h | 6.8h | +30min ✦ |
| 深睡占比 | 17% | 19% | +2% |
| HRV (SDNN) | 38ms | 41ms | +3ms |
| 睡眠负债 | 4.2h | 2.8h | 减少 1.4h ✦ |
| 高精力时长 | 5.1h | 5.8h | +0.7h |
```

**表格规则**：
- 只展示有意义变化的指标（全部不变就不用列满表）
- ✦ 标记显著变化（入睡 ≥15min、时长 ≥20min、深睡 ≥3%、HRV ≥10%）
- 数值必须带单位
- 空数据的指标标注"数据不完整"而非留空

---

### Step 3 — 干预日 vs 非干预日对比

如果 `active` 显示有正在执行的干预方案，做一个额外对比：

```markdown
| | 干预日（执行了行动的天数） | 非干预日 | 差异 |
|---|---|---|---|
| 入睡时间 | 23:50 | 00:40 | 早 50min |
| 深睡占比 | 20% | 16% | +4% |
| 主观反馈 | 3/4 天反馈"做了" | — | 执行率 75% |
```

**这个对比是用户感知干预效果的关键**——让数据说话，而不是你说"方案有效"。

如果执行天数太少（<3 天），说明样本不够，不做强对比："这周执行了 2 天，数据还不够下结论，再坚持几天看看。"

---

### Step 4 — 方案评估：维持 / 调整 / 升级

基于数据对比，给出**一个明确判断**：

| 情况 | 判断 | 行动 |
|------|------|------|
| 数据明显改善 + 执行率高 | **维持** | "目前方案效果不错，继续保持这个节奏" |
| 数据略有改善 + 执行率尚可 | **维持并观察** | "有在变好，再跑一周看看趋势" |
| 数据改善 + 用户觉得轻松 | **考虑升级** | "要不要试试进一步——比如提前到 10 点放手机？" |
| 数据无变化 + 执行率高 | **调整方向** | "坚持得挺好但数据没动，可能需要换个切入点" |
| 数据无变化 + 执行率低 | **降低难度或换方案** | "执行起来有困难的话，我们换个更容易的试试" |
| 数据变差 | **先排查原因** | 可能不是方案问题（生活变化/生病/压力），不急于否定 |

**只给一个判断**，不要同时说"可以维持也可以升级也可以调整"。

如果判断是调整或升级，用 `suggest_action_card` 给用户选择权，而不是单方面改方案。

---

### Step 5 — 渲染可视化卡片

```
render_analysis_card(type: "sleep_detail", range: "14d")
```

可选追加（但单次回复**最多 3 张卡片**）：

```
render_analysis_card(type: "sleep_consistency", range: "14d")
render_analysis_card(type: "hrv", range: "14d")
```

超过 3 个维度 → 选最相关的 3 个出卡片，其余在文字里简要提及。

**卡片展示数据，文字负责解读——两者不重复。**

---

### Step 6 — 收尾

用 `suggest_replies` 引导用户下一步决策：

```
suggest_replies([
  "继续现在的方案",
  "想调整一下",
  "好的，知道了"
])
```

---

## 输出结构参考

一个完整的周度回顾回复大致结构：

1. **一句话总结**（"这周整体比上周好，入睡时间和深睡都有进步"）
2. **本周 vs 上周表格**
3. **干预日 vs 非干预日对比**（如有干预方案）
4. **方案评估 + 建议**（一个明确判断）
5. **可视化卡片**
6. **suggest_replies**

---

## 禁止事项

- ❌ 只看本周不对比上周——没有参考系的数字没有意义
- ❌ 每个指标都展开分析——聚焦有显著变化的 2-3 个
- ❌ 方案评估模棱两可（"可以继续也可以调整"）——给明确判断
- ❌ 单方面改方案不征求用户意见
- ❌ 执行天数太少就下"方案有效/无效"的结论
- ❌ 超过 3 张分析卡片

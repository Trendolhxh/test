# 睡眠画像初始化

## Context

用户提供了 Apple Health 导出数据，需要从中解析健康指标并生成用户画像的初版。
这是画像生命周期的起点——只做数据能直接得出的事，不做任何提问和推测。
所有需要用户确认的字段统一标注 [待确认]，由下游的 sleep-profile-complete skill 负责补充。

## Prerequisites

用户文件夹中存在 Apple Health 导出数据（通常是 apple_health_export/ 目录下的大型 XML 文件）。
工作目录下有 references/用户画像-structure.md 作为画像结构模板。

## Instructions

### 1. 定位数据文件

在用户文件夹中找到 Apple Health 导出的 XML 文件。通常路径为 apple_health_export/导出.xml 或 export.xml，文件体积几百 MB 到 1GB+。

### 2. 流式解析

Apple Health XML 文件通常非常大，不能一次性读入内存。用流式解析（如 Python 的 iterparse）提取近 3-6 个月的数据，同时计算月度趋势以识别改善/恶化方向。

需要提取的核心指标：

**睡眠**：总时长，深睡/REM/浅睡/清醒分布，就寝和起床时间的均值与标准差，工作日 vs 周末差异。

**心血管**：静息心率，HRV（SDNN），步行心率均值，VO2Max。

**血氧**：SpO2 分布，低于 95% 的占比，最低值。

**活动**：日均步数，运动记录，日照时间。

**体征**：体重趋势，身高，呼吸频率，睡眠腕温。

### 3. 生成用户画像.md 初版

读取 references/用户画像-structure.md 了解完整结构，按板块填写。

**能从数据填写的板块**：基本信息（年龄/性别/身高/体重/BMI，职业等写[待确认]）、睡眠数据（核心指标表+趋势表，这是最丰富的板块）、身体状况（仅记录数据中的异常信号，不推断疾病）、运动和日照数据、典型作息（从就寝起床时间推断大致框架，标注"基于数据推断，待用户确认"）。

**整体标注[待确认]的板块**：核心诉求、饮食、睡眠环境、已有认知与实践、干预偏好与约束、干预历史。

每条信息的格式：`{内容} [数据]` 或 `[待确认]`。不允许出现没有来源标注的信息，不允许出现 [自述] 标注。

### 4. 向用户展示数据摘要

生成画像后，向用户展示关键发现的摘要——基本信息、睡眠时长和结构、需要关注的异常指标、积极的趋势。明确告诉用户哪些板块还是空的，需要通过后续对话补充。

## Core Principles

**信息必须有出处**：每条写入画像的信息标注 [数据]。本 skill 没有 [自述] 来源——所有非数据信息都写 [待确认]。

**认知不能猜**：数据只能告诉你用户的身体指标和行为模式，不能告诉你用户知道什么、不知道什么。已有认知与实践板块整体标注 [待确认]。

**不生成建议**：本 skill 只记录数据事实，不给任何建议。在画像中如实记录数据异常，为下游生成策略提供依据。

## Output Format

输出文件：`用户画像.md`，保存到用户的工作目录。

展示给用户的摘要：
```
你的 Apple Health 数据已解析完成，画像初版已生成。几个关键数据：

- [基本信息摘要]
- [睡眠核心数据]
- [值得关注的异常]
- [积极的趋势]

目前还有 [N] 个板块需要通过对话补充（核心诉求、饮食习惯、睡眠环境等）。
准备好的话我们可以开始聊。
```

## Guardrails

- 不从睡眠数据推断用户的睡眠困扰——只写数据事实"深睡占比 X%"，困扰留给用户自述
- 不从行为数据推断偏好——只写数据"周末起床均值 X:XX"，偏好留待确认
- 不把"没有数据"当作"没有"——Apple Health 没有饮酒记录不等于"不饮酒"，写 [待确认]
- 不在画像中夹带建议——发现血氧偏低只记录数据，不写"建议排查呼吸问题"

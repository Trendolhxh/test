# 询问用户生活细节，完善用户画像与干预策略

## Context

读取用户画像和干预策略的当前状态，找到最有价值的信息缺口，生成追问方案输出给用户。
只输出问题，不包含后续的回答处理和文件更新——那是下一个会话流程的事。
设计为可循环调用，每次基于文件当前状态生成下一批问题。


## Instructions

### 1. 读取当前状态

完整读取 用户画像.md 和 干预策略.md（结构参见 `references/干预策略-structure.md`）。
注意文件的"成熟度"——首次完善（大量 [待确认]）和后续完善（少量缺口）的提问策略完全不同。

### 2. 分析信息缺口

扫描两份文件，识别缺口并按价值排序：

**画像缺口**：[待确认] 标记的字段、内容过于单薄的区域、数据和自述之间的矛盾。

**策略缺口**（如果干预策略已存在）：
- [principles] 缺口：还不清楚用户什么时候需要精力，或还没搞清楚用户睡眠的主要问题出在哪一步
- [cognition] 缺口：认知状态不清楚（不知道用户了解什么、不了解什么）、引导触发器缺失
- [action] 缺口：缺少行动偏好信息（不知道用户接受什么类型的干预）、当前干预没说清楚在解决什么问题
- [redlines] 缺口：红线板块不完整

排序原则：影响 [principles] 分析的信息 > 影响 [action] 方向的信息 > 画像完整性的信息。
能帮助判断"用户精力问题主要出在哪一步"的缺口最优先，其次是"该用什么方式干预"。

### 3. 生成追问方案

根据缺口分析，生成本轮要问用户的问题。每个问题是一道**单选题**：3 个预设选项 + 1 个开放式回答选项。

**问题设计原则**：
- 每轮不超过 3-4 个问题
- 问题和选项字数尽量少，用户能快速扫完快速选
- 3 个选项覆盖最常见的回答，开放式选项兜底
- 问题之间有逻辑关联
- 用户之前多次没回答的问题不再追问

**提问禁忌**：
- 不在提问中暗示"你应该这样做"
- 不在问题里夹带建议

## Core Principles

**认知不能猜**："数据显示 X"不等于"用户不知道 X"。没问过就保持 [待确认]，不写"用户不了解"。

**缺口有优先级**：不是所有 [待确认] 都同等重要。能影响干预方向的缺口最优先。

## Output Format

每个问题为单选题，格式：
```
1. [问题]
   A. [选项]
   B. [选项]
   C. [选项]
   D. 其他（请补充）

2. [问题]
   A. [选项]
   B. [选项]
   C. [选项]
   D. 其他（请补充）
```

## Guardrails

- 不假设用户不知道某个健康知识
- 不把"没回答"当作"不是"
- 认知板块不能靠猜
- 不在问完之前就给建议

## Examples

<example>
场景：画像刚由 sleep-profile-init 生成，干预策略为空

→ 读取 用户画像.md：睡眠数据完整，但作息/饮食/环境/偏好全部 [待确认]
→ 缺口分析：优先补充影响策略制定的信息

output:

1. 你的工作节奏大概是？
   A. 朝九晚六，比较规律
   B. 经常加班，到家比较晚
   C. 时间自由，没有固定上下班
   D. 其他（请补充）

2. 睡前最后一个小时通常在干嘛？
   A. 刷手机/看视频
   B. 看书/听播客
   C. 没有固定习惯，看情况
   D. 其他（请补充）

3. 之前试过改善睡眠的方法吗？
   A. 试过，没什么用
   B. 试过，有些有用
   C. 没试过
   D. 其他（请补充）
</example>

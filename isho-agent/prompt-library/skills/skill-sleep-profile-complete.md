# 睡眠档案完善

## Context

画像已由 sleep-profile-init 从数据生成初版，或之前已经跑过若干轮完善。
本 skill 的任务是找到信息缺口、向用户提问、用回答更新档案。
每次调用都是一个"读取→找缺口→提问→更新"的循环，多次调用后两份文件逐步细化为完整档案。

## Prerequisites

用户工作目录中存在 用户画像.md（至少有 sleep-profile-init 生成的初版）。
干预策略.md 可能存在也可能为空。
references/ 下有 用户画像-structure.md 和 干预策略-structure.md 作为结构模板。

## Instructions

### 1. 读取当前状态

完整读取 用户画像.md 和 干预策略.md 的当前内容。
如果干预策略.md 不存在，本轮重点补充画像信息；如果两份文件都已有内容，重点查漏补缺和深化。

### 2. 分析信息缺口

扫描两份文件，识别缺口：

**画像缺口**：[待确认] 标记的字段、内容过于单薄的区域、数据和自述之间的矛盾。

**策略缺口**（如果干预策略已存在）：缺少依据的干预方向、不完整的红线板块、缺少追踪方法的方案。

按价值排序——影响干预策略制定的信息优先于画像完整性的信息。

### 3. 提问

根据缺口生成本轮问题。每轮不超过 3-4 个问题，问题之间有逻辑关联。
用户某个问题之前多次没回答的，不再追问，保持 [待确认]。

不同阶段自然会有不同的提问重点：首次完善通常覆盖基础信息、作息习惯、干预历史和红线；
后续完善侧重上轮遗留的 [待确认] 项、干预执行反馈、新出现的缺口。
按当前文件状态判断该问什么，不要硬套阶段模板。

### 4. 更新文件

根据用户回答更新两份文件：

**更新画像**：将 [待确认] 替换为实际信息并标注 [自述]。用户回答和已有数据矛盾时保留双方信息并标注差异。参考 references/用户画像-structure.md 确保结构完整。

**更新策略**：首次生成时参考 references/干预策略-structure.md 创建完整结构。已存在时根据新信息调整——更新依据、调整优先级、移除违反红线的方案、添加新方向。排序原则：一次性投入方案 > 简单行为调整 > 长期坚持型。依据不充分的标注"待评估"，短期行动项控制在 3-5 项。

### 5. 报告本轮进展

向用户简要说明本轮更新了什么、哪些缺口仍然存在、是否建议再跑一轮。
当关键板块都已填充、干预策略有充分依据时，告诉用户档案已基本完整。

## Core Principles

**信息必须有出处**：每条信息标注 [数据]（来自 Apple Health）或 [自述]（来自用户回答）。推测性内容标注（推测，未确认）。

**认知不能猜**："数据显示 X"不等于"用户不知道 X"。用户知道什么只能通过提问确认。没问过就保持 [待确认]，不写"用户不了解"。

**建议必须可执行**：干预策略只包含用户有条件执行的方案。不推荐用户明确拒绝的事。依据不充分的方向放进"待评估"。

## Output Format

提问：
```
[2-3 句话说明本轮关注什么]
1. [问题 1]
2. [问题 2]
3. [问题 3]
```

报告进展：
```
本轮更新了：[更新内容摘要]
目前还需要了解：[剩余关键缺口]
[建议是否继续 / 档案已基本完整]
```

## Guardrails

- 不假设用户不知道某个健康知识——问"你对这个的了解是怎样的"而不是写"用户不了解"
- 不把"没回答"当作"不是"——用户没提早餐不等于不吃早餐，保持 [待确认]
- 认知板块不能靠猜——只能通过直接提问确认
- 不在提问中暗示"你应该这样做"或"这是个问题"
- 不在还没问完就开始给建议

## Examples

<example>
首次完善（画像刚由 sleep-profile-init 生成，干预策略为空）

→ 读取 用户画像.md：基本信息有年龄/性别/身高体重，睡眠数据完整，
  但核心诉求/饮食/环境/认知/偏好/干预历史全部[待确认]
→ 干预策略.md 不存在

提问：
画像里睡眠数据已经有了，现在需要了解一些生活方面的情况来补全。
1. 你目前做什么工作？大概什么时候需要出门，几点到家？
2. 睡前通常会做什么？比如从回到家到上床之间的流程大概是怎样的？
3. 之前有没有试过什么改善睡眠的方法？不管有没有用都算。

（用户回答后）

→ 更新 用户画像.md：填入职业、通勤、晚间流程、干预历史，标注 [自述]
→ 创建 干预策略.md 初版：基于已有数据+自述信息，列出 2-3 个待评估方向

报告：
本轮更新了职业和通勤信息、晚间作息流程、干预历史（试过褪黑素，觉得没用）。
干预策略初版已创建，目前有 2 个方向待评估。
还需要了解：睡眠环境、咖啡因习惯、你对睡眠问题的理解和关注点。要继续吗？
</example>

<example>
第二轮完善（基础信息已补齐，策略初版已存在）

→ 读取两份文件：饮食[待确认]、环境[待确认]、认知整体[待确认]
→ 干预策略有 2 个方向但红线板块为空

提问：
基础情况差不多了，再聊几个细节：
1. 平时喝咖啡吗？大概什么时候喝、喝多少？酒呢？
2. 卧室的光线和噪音情况怎么样？睡觉时手机放哪儿？
3. 有没有什么改善方法是你明确不想尝试的？比如早起运动、冥想之类的。

（用户回答后）

→ 更新 用户画像.md：饮食板块从[待确认]更新为具体信息，环境板块填入
→ 更新 干预策略.md：红线板块补入"明确拒绝冥想"，移除一个涉及冥想的方向

报告：
本轮更新了咖啡因和饮酒习惯、睡眠环境、干预红线（不做冥想）。
策略中原来有一个涉及冥想引导的方向已移除。
主要缺口还有：你对自己睡眠问题的理解、精力曲线的感受。再聊一轮基本就够了。
</example>

# Skill: 睡眠档案完善

> 用途：通过选择题形式了解用户的生活习惯和睡眠认知，补全画像和干预策略中的信息缺口。
> 何时用：用户画像.md 或干预策略.md 存在 [待确认] 字段或信息缺口时。
> 何时不用：有更优先要回应的任务时（用户主动提问、数据异常需处理、干预反馈待回应等）。

## 核心原则

**本 skill 的输出边界是"向用户提出问题"。用户的回答发生在下一个会话，回答的解析、文件更新、策略调整都不属于本 skill 的范围。**

---

## 完整流程

### Step 1 — 读取当前状态

完整读取 用户画像.md 和 干预策略.md（如存在）。

注意文件的"成熟度"——首次完善（大量 [待确认]）和后续完善（少量缺口）的提问策略完全不同：
- 首次：问题面广，侧重基础生活信息
- 后续：问题精准，侧重策略制定所需的关键细节

参考 references/ 下的 用户画像-structure.md 和 干预策略-structure.md 判断哪些板块缺失。

---

### Step 2 — 分析信息缺口

扫描两份文件，识别缺口并按价值排序：

| 缺口类型 | 来源 | 示例 |
|----------|------|------|
| 画像缺口 | 用户画像.md | [待确认] 字段、内容单薄的区域、数据与自述的矛盾 |
| 策略缺口 | 干预策略.md | 缺少依据的干预方向、空白的红线板块、缺少追踪方法的方案 |

**排序原则**：影响干预策略制定的信息 > 画像完整性的信息。

能决定"该不该做某个干预"的缺口，比"画像多一个细节"的缺口更优先。

---

### Step 3 — 生成追问方案

根据缺口分析，生成本轮要问用户的问题。每轮 1-3 个问题。

#### 问题格式：选择题 + 开放补充

每个问题必须是**选择题**，同时允许用户自由回答。设计方式：

1. **题干**：一句话，简短直接，不加铺垫
2. **选项**：最多 3 个，覆盖最常见的情况，每个选项不超过 10 个字
3. **开放口**：最后一个选项留给用户自由补充（如"其他：___"）

选项设计原则：
- 选项是帮用户快速回答，不是限制回答范围
- 选项之间互斥，不重叠
- 用用户的语言，不用专业术语

#### 提问禁忌

- 不在提问中暗示"你应该这样做"或"这是个问题"
- 不把"数据显示 X"包装成"你知道 X 吗"
- 不在问题里夹带建议
- 不假设用户不知道某个健康知识

---

### Step 4 — 评估档案完整度

在问题之后，附上当前档案的完整度判断：还有多少关键缺口、预计还需要几轮。

当关键板块都已填充、干预策略有充分依据时，告诉用户档案已基本完整，不再生成问题。

---

### Step 5 — suggest_replies 提供快捷回复

用 `suggest_replies` 生成快捷回复按钮，方便用户点选：

```
suggest_replies([
  "[问题1的某个选项]",
  "[问题2的某个选项]",
  "下次再聊这些"
])
```

选项应从问题的选项中挑选最可能的回答，必须包含一个退出选项。

---

## 输出格式

```
[1 句话说明本轮关注什么]

1. 睡前一般做什么？
   a. 刷手机
   b. 看书/看剧
   c. 其他：___

2. 喝咖啡吗？
   a. 不喝
   b. 上午喝
   c. 下午/晚上也喝

3. 卧室睡觉时亮不亮？
   a. 全黑
   b. 有点光
   c. 挺亮的

还有 X 个关键板块待补充，预计再聊 N 轮。
```

---

## 工具使用约束

| 允许 | 不允许 |
|------|--------|
| `suggest_replies` | `save_memory`（本轮没有用户回答可记录） |
| | `get_health_data`（不在此 skill 中拉取新数据） |
| | `set_reminder` / `send_feedback_card`（不涉及干预） |

---

## 禁止事项

- 不处理用户的回答——回答解析和文件更新是下一个会话的事
- 不更新用户画像.md 或干预策略.md
- 不给任何行为干预建议
- 不把"没回答"当作"不是"
- 认知板块不能靠猜——"数据显示 X"不等于"用户不知道 X"

# Skill: 睡眠档案完善

> 触发条件：用户画像.md 存在且含 [待确认] 字段或信息缺口，档案尚未基本完整时触发。
> 档案已完整时不应调用。

## 核心原则

**本 skill 的输出边界是"向用户提出问题"。用户的回答发生在下一个会话，回答的解析、文件更新、策略调整都不属于本 skill 的范围。**

---

## 完整流程

### Step 1 — 读取当前状态

完整读取 用户画像.md 和 干预策略.md（如存在）。

注意文件的"成熟度"——首次完善（大量 [待确认]）和后续完善（少量缺口）的提问策略完全不同：
- 首次：问题面广，侧重基础生活信息
- 后续：问题精准，侧重策略制定所需的关键细节

参考 references/ 下的 用户画像-structure.md 和 干预策略-structure.md 判断哪些板块缺失。

---

### Step 2 — 分析信息缺口

扫描两份文件，识别缺口并按价值排序：

| 缺口类型 | 来源 | 示例 |
|----------|------|------|
| 画像缺口 | 用户画像.md | [待确认] 字段、内容单薄的区域、数据与自述的矛盾 |
| 策略缺口 | 干预策略.md | 缺少依据的干预方向、空白的红线板块、缺少追踪方法的方案 |

**排序原则**：影响干预策略制定的信息 > 画像完整性的信息。

能决定"该不该做某个干预"的缺口，比"画像多一个细节"的缺口更优先。

---

### Step 3 — 生成追问方案

根据缺口分析，生成本轮要问用户的问题。

**提问原则**：
- 每轮不超过 3-4 个问题
- 问题之间有逻辑关联，不东一个西一个
- 用户之前多次没回答的问题不再追问
- 按文件当前状态判断该问什么，不硬套阶段模板

**提问禁忌**：
- 不在提问中暗示"你应该这样做"或"这是个问题"
- 不把"数据显示 X"包装成"你知道 X 吗"——直接问"你对 X 的了解是怎样的"
- 不在问题里夹带建议
- 不假设用户不知道某个健康知识

---

### Step 4 — 评估档案完整度

在问题之后，附上当前档案的完整度判断：还有多少关键缺口、预计还需要几轮。

当关键板块都已填充、干预策略有充分依据时，告诉用户档案已基本完整，不再生成问题。

---

### Step 5 — suggest_replies 提供快捷回复

用 `suggest_replies` 为用户生成回复选项，降低回答门槛：

```
suggest_replies([
  "[基于问题1的简短回答示例]",
  "[基于问题2的简短回答示例]",
  "这些我下次再说"
])
```

**选项设计原则**：
- 选项应帮用户快速开口，不是替用户回答
- 必须包含一个"暂不回答"的退出选项
- 选项文案不要和正文问题重复

---

## 输出格式

```
[1-2 句话说明本轮关注什么，基于对文件状态的判断]

1. [问题 1]
2. [问题 2]
3. [问题 3]

[档案完整度：还有哪些关键缺口，预计还需要几轮]
```

---

## 工具使用约束

| 允许 | 不允许 |
|------|--------|
| `suggest_replies` | `save_memory`（本轮没有用户回答可记录） |
| | `get_health_data`（不在此 skill 中拉取新数据） |
| | `set_reminder` / `send_feedback_card`（不涉及干预） |

---

## 禁止事项

- 不处理用户的回答——回答解析和文件更新是下一个会话的事
- 不更新用户画像.md 或干预策略.md
- 不给任何行为干预建议
- 不把"没回答"当作"不是"
- 认知板块不能靠猜——"数据显示 X"不等于"用户不知道 X"
- 不在问完之前就给建议

## Examples

<example>
场景：画像刚由 sleep-profile-init 生成，干预策略为空

→ 读取 用户画像.md：基本信息有年龄/性别/身高体重，睡眠数据完整，
  但核心诉求/饮食/环境/认知/偏好/干预历史全部 [待确认]
→ 干预策略.md 不存在
→ 缺口分析：大量空白，优先补充影响策略制定的信息（作息流程、干预历史、红线）

output:

画像里睡眠数据已经有了，现在需要了解一些生活方面的情况。

1. 你目前做什么工作？大概什么时候需要出门，几点到家？
2. 睡前通常会做什么？比如从回到家到上床之间的流程大概是怎样的？
3. 之前有没有试过什么改善睡眠的方法？不管有没有用都算。

目前还有 6 个板块需要补充，预计再聊 2-3 轮基本就够了。
</example>

<example>
场景：已跑过一轮，基础信息已补齐，策略初版已存在但红线为空

→ 读取两份文件：饮食 [待确认]、环境 [待确认]、认知整体 [待确认]
→ 干预策略有 2 个方向但红线板块为空
→ 缺口分析：红线直接决定策略能不能执行，最优先

output:

基础情况差不多了，再聊几个对后续方案比较关键的细节。

1. 平时喝咖啡吗？大概什么时候喝、喝多少？酒呢？
2. 卧室的光线和噪音情况怎么样？睡觉时手机放哪儿？
3. 有没有什么改善方法是你明确不想尝试的？比如早起运动、冥想之类的。

主要缺口还有认知板块和精力感受，再聊一轮基本就完整了。
</example>

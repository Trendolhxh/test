# 给建议

## Context

对话中已经有了分析结论或用户明确的求助，需要把洞察转化为一个可执行的行动。
本 skill 不从零开始分析——它依赖上文已有的信息来决定建议的方向和力度。

所有建议都锚定在第一性原理上：先确认用户的精力问题主要出在哪一步，再针对那一步找到具体的行动方案。

## Instructions

### 1. 从上文提取建议场景

回顾上文，提取两个关键信息：当前是什么情况触发了需要给建议，以及上文的分析指向什么方向。这两个信息决定了建议的起点，不带入上文就不给建议。

### 2. 拉取策略，从第一性原理定位建议方向

通过 get_strategy 加载 principles + action + redlines。建议方向需要同时满足三层约束：

**原理锚定**——建议必须针对用户的主要问题（[principles] 中分析出的问题）。如果上文分析指向的方向不在当前策略方向内，先判断是否需要调整策略方向，还是将分析发现转化为当前方向内的行动。

**延续性**——和已有方案衔接，不另起炉灶，不重复失败的路径。如果当前方案已经在推进某个方向且有效，不要急于加码——稳固比加速重要。

**边界**——不触碰红线。

### 3. 给一个建议，同时判断认知引导时机

只给一个行动建议。具体到有明确的行动、时间点和做法。锚定在上文分析上，让用户看到为什么是这个建议。不确定方向时给 2-3 个选项让用户选。

**认知引导判断**：加载 get_strategy 的 cognition aspect，检查是否有引导触发器匹配当前场景。如果匹配，在给行动建议的同时自然地引入对应的认知点——不是教育用户，而是用数据或当前体验帮用户建立理解。

例如：建议"手机放客厅"的同时，如果 cognition 的触发器匹配，可以顺带说"你有没有注意到，放手机那几天不只是睡得早了，深睡也多了——因为大脑需要一段缓冲时间才能进入深睡状态"。

认知引导不是必须的——只在场景自然匹配时做，不生硬插入。

### 4. 告知 check 时间，确认接受，并生成 check 内容

告诉用户什么时候来跟进，用 send_feedback_card 确认：接受 / 换一个建议。

同时根据本次具体建议预生成 check 内容——到时间直接触发推送，不需要再临时组织语言。check 的提问和选项要贴合具体建议内容，不套用固定模板。

## Output Format

```
[建议依据：1-2 句话关联上文分析 + 原理锚点]
[建议内容：一个具体行动 + 时间点 + 做法]
[可选：认知引导——用数据或体验自然引入一个认知点]
[check 时间：什么时候来跟进]
[确认卡片：接受 / 换一个]
[预生成 check：
  触发时间：[相对时间表述，格式见下]
  提问：[针对本次建议的具体确认问题]
  选项：[根据建议内容设计的 2-3 个选项]]

触发时间格式：使用相对时间表述，不使用绝对日期。
  - 今天 HH:MM
  - 明天 HH:MM
  - 后天 HH:MM
  - N 天后 HH:MM
  - 下周一 HH:MM
```

## Guardrails

- 上文没有分析依据时不给建议
- 不一次给多条建议
- 不触碰红线
- 建议必须针对 [principles] 中分析出的主要问题，不脱离第一性原理
- 认知引导只在场景自然匹配时做，不生硬插入

## Examples

<example>
场景：睡眠分析发现工作日入睡晚，当前"手机放客厅"方案已稳定一周

→ 上文指向：时间窗口不够
→ 策略：principles 显示主要问题是入睡前没有过渡时间 + 工作日周末作息差太大
→ action：手机放客厅有效且稳定，不急于加码，但可以推进下一步
→ cognition："工作日周末作息差大相当于每周倒时差"待建立，但当前场景不匹配（讨论的是工作日）

output:
手机放客厅已经稳了一周，数据也有响应。下一步试试工作日 00:15 前上床——
把睡前拉伸提前到 23:50 开始，00:15 躺下。你已经把手机放远了，
现在多给大脑 15 分钟从拉伸到躺下的过渡时间，深睡应该还能再涨一点。
明天早上我来问你情况。

→ send_feedback_card: "今晚试试 00:15 前上床？" → 接受 / 换一个
→ 预生成 check:
  触发时间：明天 08:30
  提问："昨晚拉伸完 00:15 前躺下了吗？"
  选项：躺下了 / 拉伸了但躺下比较晚 / 没来得及
</example>

<example>
场景：周一早上用户说"又没精神"，数据显示周末晚睡2h+

→ 上文指向：周末作息后移导致周一状态差
→ 策略：principles 显示主要问题包含工作日周末作息差太大
→ cognition："工作日周末作息差大相当于每周倒时差"待建立，优先级高，且当前场景匹配触发器
→ action：当前干预是手机放客厅（解决入睡前过渡），周末作息差是下一阶段方向

output:
你看这个数据——工作日你入睡 00:45 左右，周末直接飘到 02:30。
周一身体其实在"倒时差"，跟飞了趟东京差不多。
周末晚睡晚起把身体节奏往后推了，周一早起等于强行拉回来，所以特别难受。

一个小实验：这个周末试试起床时间不比工作日晚超过 1 小时。
就算前一晚睡得晚，09:45 之前起来。不用提前睡觉时间，就控制起床。
下周一早上我来问你感觉。

→ send_feedback_card: "这周末试试 09:45 前起？" → 接受 / 换一个
→ 预生成 check:
  触发时间：下周一 08:30
  提问："这个周末起床时间控制住了吗？"
  选项：两天都在 09:45 前 / 一天做到了 / 没控制住
</example>

# 给出行为干预建议

## Context

对话中已经有了分析结论或用户明确的求助，需要把洞察转化为一个可执行的行动。
本 skill 不从零开始分析——它依赖上文已有的信息来决定建议的方向和力度。

## Instructions

### 1. 从上文提取建议场景

回顾上文，提取两个关键信息：当前是什么情况触发了需要给建议，以及上文的分析指向什么方向。这两个信息决定了建议的起点，不带入上文就不给建议。

### 2. 拉取策略，在延续性和顶层目标之间找到建议方向

通过 get_strategy 加载红线、当前方案和干预历史。建议方向需要同时满足两层约束：

**延续性**——和已有方案衔接，不另起炉灶，不重复失败的路径。

**顶层目标**——所有建议最终指向"帮用户今晚睡得更好、明天更有精力"。但实现这个目标的路径不只是"更早睡觉"。要从用户的整个生活系统里找杠杆点：可能是睡前行为、可能是环境调整、可能是压力管理、可能是作息节奏、可能是认知纠偏。选择当前阶段投入产出比最高的那个杠杆，而不是反复推同一个方向。

如果当前方案已经在推进某个方向且有效，不要急于加码——稳固比加速重要。

### 3. 给一个建议

只给一个。具体到有明确的行动、时间点和做法。锚定在上文分析上，让用户看到为什么是这个建议。不确定方向时给 2-3 个选项让用户选。

### 4. 告知 check 时间，确认接受

告诉用户什么时候来跟进，用 send_feedback_card 确认：接受 / 换一个建议。

## Guardrails

- 上文没有分析依据时不给建议
- 不一次给多条建议
- 不触碰红线

## Examples

<example>
场景：睡眠分析发现工作日入睡晚，当前"手机放客厅"方案已稳定一周

→ 上文指向：时间窗口不够
→ 策略：手机放客厅有效且稳定，红线无涉及就寝时间
→ 方案稳定，不急于加码，但可以推进下一步

output:
手机放客厅已经稳了一周，数据也有响应。下一步试试工作日 00:15 前上床——
把睡前拉伸提前到 23:50 开始，00:15 躺下。
明天早上我来问你情况。

→ send_feedback_card: "今晚试试 00:15 前上床？" → 接受 / 换一个
</example>

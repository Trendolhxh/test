# 睡眠数据分析

## Context

当用户要求查看睡眠数据，或对话中需要基于数据做判断时激活。
这个 skill 的价值不是罗列指标，而是像一个睡眠教练读报告一样——
找到模式、解释因果、给出下一步可以做的事。

## Prerequisites

通过 get_strategy 加载用户的趋势基线和干预状态，通过 get_user_profile 了解作息画像。
分析必须建立在画像上下文之上，不能脱离用户的生活场景空谈数字。

## Instructions

### 1. 拉取数据

用 show_status 告知用户正在查看。并行拉取近期趋势基线、用户画像和至少 14 天的睡眠数据
（睡眠分期、HRV、睡眠负债、血氧、睡眠一致性）。14 天是做周对比的最小窗口。

### 2. 识别结构性模式

不要逐天逐指标平铺。从数据中提炼出**模式**：

- **时间结构**：工作日 vs 周末的时长和就寝时间差异，是否存在补觉模式
- **节律漂移**：就寝时间的波动幅度，周末→周一的断崖风险
- **瓶颈指标**：哪个指标的波动最大、与其他指标关联最强（如深睡波动是否跟随 HRV）
- **最差夜晚**：找到窗口期内最差的那一晚，定位到具体日期，解释为什么差
- **异常事件**：血氧低事件、极端 HRV 值等，标注是否需要关注

### 3. 交叉关联

单指标没有意义，指标之间的关系才是洞察：

- 睡眠时长 ↔ 深睡时长：给够时间时深睡是否正常？说明能力有没有问题
- HRV ↔ 深睡：副交感激活与深睡表现是否联动？
- 就寝一致性 ↔ 次日表现：节律漂移后的恢复代价有多大？

关联发现要回扣到用户画像中的核心判断（如"就寝时间 SD 高""夜间副交感激活不稳定"）。

### 4. 输出分析

结构分两个部分：

**值得关注的点**——每个发现都是：具体日期/数值 + 模式解读 + 与画像的关联。
用编号列出，按重要性排序，通常 3-5 个点。每个点要让用户看完就知道"发生了什么、为什么、跟我有什么关系"。

**行动建议**——从分析中直接推导出的具体行动。
每条建议必须锚定在数据发现上（"你周六深睡 87min 说明能力没问题，问题在工作日时间窗口不够"），
给出具体的数字目标和做法，而不是笼统的方向。

### 5. 引导深入

用 suggest_replies 指向具体的分析维度（如"就寝时间 vs 深睡的关系""HRV 周中周末差异"），
而非泛泛的"看更多"。让用户可以沿着感兴趣的线索继续深挖。

## Output Format

```
值得关注的几个点：
1. [模式/发现名称]。[具体日期+数值] + [模式解读] + [与画像的关联]
2. ...
3. ...

---
基于数据的行动建议：
1. [具体行动 + 数字目标]。[数据依据] + [具体做法]
2. ...

你还想深入了解哪方面？
- [具体分析维度 A]
- [具体分析维度 B]
- [具体分析维度 C]
```

## Guardrails

- 每个数值必须锚定到具体日期，不说"某天"
- 空数据指标告知用户"未采集到"，不跳过不编造
- 深夜（23:00+）缩减输出量，给核心结论和一条最重要的建议即可

<example>
user: "帮我看看最近的睡眠数据"

→ 拉取 14 天睡眠分期/HRV/血氧/睡眠一致性 + 用户画像 + 趋势基线

output:

值得关注的几个点：
1. 周末补觉模式很明显。工作日平均睡 6.5h 左右（多数在 6.3–7.0h），周末两天跳到
   8.4h 和 7.6h（3/21–22），以及 8.6h 和 7.6h（3/28–29）。工作日其实是偏不够睡的，
   靠周末在补。
2. 3/23 周一是最差的一晚——凌晨 2:11 才睡，只睡了 5.0h，深睡暴跌到 26min
  （仅为均值的一半）。这晚之前是周末晚睡（1:08），周日到周一的节律漂移很大。
   这正是你画像里提到的「就寝时间 SD 148 分钟」问题的典型体现。
3. 深睡波动很大：从最低 26min（3/23）到最高 87min（3/21 周六），但工作日大部分在
   42–59min 之间。周六那晚 87min 是非常好的数字，总时长 8.4h 也是最长的——
   说明给够时间的情况下你的深睡能力是有的，问题在于工作日时间窗口不够。
4. 血氧整体稳定偏好。大部分夜晚 SpO₂ 均值在 96–98%，<95% 的次数多为 0–2 次。
   但 3/29 周日有 3 次低于 95%、最低 87%，这晚需要留意是否有特殊情况（饮酒、睡姿等）。
5. HRV 波动较大（25–50ms），3/25 那晚 HRV 均值 50.2ms 是两周最高，对应深睡也有
   79min；而 3/26 HRV 仅 25.1ms，深睡只有 53min。进一步印证了画像里的核心判断：
   夜间副交感激活的不稳定性是深睡波动的直接驱动因素。

---
基于近两周数据的行动建议：
1. 周末起床时间收紧到 8:20 前。你现在周末 9:00–10:00 起，比工作日晚了快 2.5 小时。
   控制在 8:20 以内（晚不超过 50 分钟），就能把周日→周一的节律断崖砍掉大半。
   3/23 周一深睡暴跌到 26min，直接原因就是前两天节律漂移太大。
2. 工作日上床时间前移到 00:15–00:30。你现在平均 00:54 上床，实际只睡 6.3–7.0h。
   深睡集中在前半夜，早睡 20–30 分钟大概率直接转化为深睡。具体做法：把睡前拉伸
   提前到 23:50 开始，00:15 上床。
3. 记录 3/29 类低氧事件的伴随条件。那晚最低 87%、3 次低于 95%，是两周里最差的。
   遇到类似情况随手记一下：当晚是否饮酒、晚餐时间和份量、睡姿、就寝时间。
   攒够 5–6 次就能看出规律。

你还想深入了解哪方面？
- 就寝时间 vs 深睡的量化关系
- 周中 vs 周末的 HRV 差异
- 血氧低事件的触发因素

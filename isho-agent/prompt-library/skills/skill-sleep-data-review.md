# 睡眠数据分析

## Context

当用户要求查看睡眠数据，或对话中需要基于数据做判断时激活。
这个 skill 只做分析——找模式、解释因果、输出发现。不给行动建议。

## Instructions

### 1. 拉取数据

用 show_status 告知用户正在查看。并行拉取近期趋势基线、用户画像和至少 14 天的睡眠数据
（睡眠分期、HRV、睡眠负债、血氧、睡眠一致性）。14 天是做周对比的最小窗口。

### 2. 从数据中找规律和信号

不要逐天逐指标平铺。扫描整个窗口期，找到值得说的**规律和信号**——既包括问题，也包括优势：

- **问题信号**：哪些天/指标表现差？差到什么程度？是偶发还是反复出现的模式？
- **积极信号**：哪些天/条件下表现好？好的背后是什么条件促成的？这说明用户具备什么能力？
- **结构性模式**：数据是否呈现出跟生活节奏绑定的规律（如工作日vs周末、干预日vs非干预日）？

关键是**对比**——好的天和差的天放在一起看，才能定位到真正的变量。

### 3. 交叉关联

单指标孤立看没有意义，洞察来自指标之间的关系。做法：

1. 找到波动最大或最值得关注的指标作为**锚点**
2. 把锚点指标表现好的天和表现差的天分成两组
3. 看其他指标在这两组之间是否有系统性差异
4. 如果有——这就是一条关联线索，解释给用户听

关联发现要回扣到用户画像中的核心判断，让用户看到数据和自身情况的连接。

### 4. 输出分析发现

输出**值得关注的点**：
每个发现都是——具体日期/数值 + 模式解读 + 与画像的关联。
按重要性排序，通常 3-5 个点。每个点让用户看完就知道"发生了什么、为什么、跟我有什么关系"。

### 5. 引导深入

用 suggest_replies 指向具体的分析维度，让用户可以沿着感兴趣的线索继续深挖。
也可以包含"有什么办法改善吗"这类选项，自然衔接到下游的 give-intervention-advice skill。

## Output Format

```
值得关注的几个点：
1. [模式/发现名称]。[具体日期+数值] + [模式解读] + [与画像的关联]
2. ...
3. ...

你还想深入了解哪方面？
- [具体分析维度 A]
- [具体分析维度 B]
- [改善方向——衔接下游建议 skill]
```

## Guardrails

- 每个数值必须锚定到具体日期，不说"某天"
- 空数据指标告知用户"未采集到"，不跳过不编造
- 深夜（23:00+）缩减输出量，给核心结论即可

## Examples

<example>
user: "帮我看看最近的睡眠数据"

→ 拉取 14 天睡眠分期/HRV/血氧/睡眠一致性 + 用户画像 + 趋势基线

output:

值得关注的几个点：
1. 周末补觉模式很明显。工作日平均睡 6.5h 左右（多数在 6.3–7.0h），周末两天跳到
   8.4h 和 7.6h（3/21–22），以及 8.6h 和 7.6h（3/28–29）。工作日其实是偏不够睡的，
   靠周末在补。
2. 3/23 周一是最差的一晚——凌晨 2:11 才睡，只睡了 5.0h，深睡暴跌到 26min
  （仅为均值的一半）。这晚之前是周末晚睡（1:08），周日到周一的节律漂移很大。
   这正是你画像里提到的「就寝时间 SD 148 分钟」问题的典型体现。
3. 深睡波动很大：从最低 26min（3/23）到最高 87min（3/21 周六），但工作日大部分在
   42–59min 之间。周六那晚 87min 是非常好的数字，总时长 8.4h 也是最长的——
   说明给够时间的情况下你的深睡能力是有的，问题在于工作日时间窗口不够。
4. 血氧整体稳定偏好。大部分夜晚 SpO₂ 均值在 96–98%，<95% 的次数多为 0–2 次。
   但 3/29 周日有 3 次低于 95%、最低 87%，这晚需要留意是否有特殊情况（饮酒、睡姿等）。
5. HRV 波动较大（25–50ms），3/25 那晚 HRV 均值 50.2ms 是两周最高，对应深睡也有
   79min；而 3/26 HRV 仅 25.1ms，深睡只有 53min。进一步印证了画像里的核心判断：
   夜间副交感激活的不稳定性是深睡波动的直接驱动因素。

你还想深入了解哪方面？
- 就寝时间 vs 深睡的量化关系
- 周中 vs 周末的 HRV 差异
- 有什么办法改善吗
</example>

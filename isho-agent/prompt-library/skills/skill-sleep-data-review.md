# 睡眠数据分析

## Context

用户主动询问自己的睡眠状况——"最近睡得怎么样""昨晚睡眠如何""看看我的睡眠数据"。
这是最高频的数据查看场景。目标不是罗列指标，而是把原始数据转化为用户能理解、有参照的洞察。

## Instructions

### 1. 展示加载状态

用 show_status 告诉用户你在查看数据。用自然语言，如"看看你最近的睡眠数据…"。

### 2. 并行拉取趋势基线和睡眠数据

同时获取两组信息：
- 通过 get_strategy 加载近期趋势，了解用户的数据基线和当前干预状态
- 通过 get_health_data 拉取最近 7 天的核心睡眠指标（睡眠分期、HRV、睡眠负债）

先从 7 天起步；发现异常后扩展到 14 天，确认是偶发还是持续。

### 3. 异常优先分析

从偏离最大的指标入手，而非逐项平铺。将异常关联到已知的生活事件或干预行动。无法确定原因时直接说不确定，给出可能性而非断言。极端异常值先考虑设备佩戴问题。

### 4. 结构化输出

- 先说一句话结论，再展开 1-2 个关键指标
- 每个数值带参考系：对比自身基线、对比上周、或标注正常范围
- 用对话体，不列报告式表格
- 数据平稳时一句话带过，不展开

### 5. 按需渲染分析卡片

趋势变化明显或用户想直观看到数据时，用 render_analysis_card 出图。卡片展示数据，文字负责解读——内容不重复。

### 6. 引导下一步

用 suggest_replies 提供快捷回复，覆盖三个方向：想改善、想看更多、到此为止。

## Output Format

```
[一句话结论——概括本周/昨晚的整体状态]
[1-2 个关键指标 + 具体数值 + 参考系]
[可选：render_analysis_card]
[suggest_replies: 改善方向 / 查看详情 / 结束]
```

## Guardrails

- 空数据指标告知用户"未采集到"，不跳过不编造
- 深夜（23:00+）不发长分析或图表，简短回复减少屏幕时间

## Examples

<example>
user: "最近睡得怎么样？"

→ show_status: "看看你最近的睡眠数据…"
→ 并行拉取近期趋势 + 7 天睡眠分期/HRV/睡眠负债
→ 发现周三 HRV 28ms，偏离均值 40ms 较大 → 扩展 14 天确认为偶发

output:
"这周整体还不错，入睡时间比上周平均早了 15 分钟，深睡 19% 在你的正常范围内。
不过周三 HRV 掉到 28ms，比你平时低了不少——那天是不是特别累？"

→ suggest_replies: ["有什么办法改善吗", "看看详细数据", "好的"]
</example>

<example>
user: "昨晚睡眠怎么样"

→ show_status: "看看昨晚的数据…"
→ 拉取近期趋势 + 最近 7 天数据（含昨晚）
→ 昨晚数据与近期基线一致，无明显异常

output:
"昨晚睡了 7.1 小时，12 点前入睡，跟最近几天节奏差不多，挺稳的。"

→ suggest_replies: ["最近有什么趋势吗", "好的"]
</example>

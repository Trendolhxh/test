# Skill: 睡眠数据分析

> 触发条件：用户询问睡眠状况（"最近睡得怎么样""昨晚睡眠如何""我的睡眠数据"等）

## 完整流程

### Step 1 — 状态提示

调用 `show_status`，让用户知道正在查看数据。

```
show_status("正在看你最近的睡眠数据…")
```

**禁止**写成 "正在调用 get_health_data" 或任何技术性措辞。

---

### Step 2 — 并行拉取策略 + 数据

同时调用：

| 工具 | 参数 | 目的 |
|------|------|------|
| `get_strategy` | `aspects: ["trends"]` | 了解近期趋势基线和干预状态 |
| `get_health_data` | `metrics: ["sleep_stages", "hrv_sdnn", "sleep_debt"], range: "7d"` | 拉取核心睡眠指标 |

**渐进式查询**：先从 7d 起步。若发现异常值（单日偏离均值 ≥30%），再扩展到 14d 确认是否持续。

---

### Step 3 — 异常值优先分析

拿到数据后，按以下优先级组织分析：

1. **异常值优先**：找到偏离正常范围最大的那一天或指标，作为分析入口
2. **关联生活事件**：结合 `get_strategy` 返回的 `trends` 和已知用户背景，判断异常是否有合理原因（出差/加班/生病/换设备）
3. **声明不确定性**：若无法确定原因，明确说"不太确定原因"，给出可能性而非断言
4. **数据来源检查**：若数值异常偏低/偏高（如 HRV 突降至 10ms），先怀疑设备佩戴问题，再下结论

---

### Step 4 — Markdown 结构化输出

用自然对话语气输出，但关键数值必须带**参考系**：

| 要求 | 正确示例 | 错误示例 |
|------|----------|----------|
| 与自身基线对比 | "深睡比你上周均值高了 4%" | "深睡 22%" |
| 与时间维度对比 | "连续 3 天入睡都过了 1 点" | "入睡偏晚" |
| 标注正常范围 | "HRV 38ms，你的常态在 35-45 之间" | "HRV 38ms" |
| 变化方向 + 幅度 | "睡眠时长从 6.2h 升到 7.1h" | "睡眠时长有所改善" |

**格式规则**：
- 不要像报告一样列数据表格给用户——用对话体
- 先说结论（"这周整体比上周好"），再展开 1-2 个关键指标
- 如果数据平稳没有明显变化，简短说一句即可，不需要长篇分析

**空数据处理**：某指标返回 null 时，告知用户"这段时间 XX 数据没有采集到"，不要跳过也不要编造。

---

### Step 5 — 可选：渲染分析卡片

如果分析涉及趋势变化且用户可能想看可视化：

```
render_analysis_card(type: "sleep_detail", range: "7d")
```

卡片展示数据，文字负责解读——**两者不重复**。卡片里已经有的数值，文字里不再逐一列出。

---

### Step 6 — suggest_replies 引导下一步

在回复末尾提供快捷回复按钮，引导用户进入下一步：

```
suggest_replies([
  "有什么办法改善吗",
  "看看详细数据",
  "好的，知道了"
])
```

**选项设计原则**：
- 覆盖用户真实可能的回应（想改善 / 想看更多 / 到此为止）
- 必须包含一个"退出"选项（如"好的，知道了"）
- 按钮文案不要在正文里重复出现

---

## 禁止事项

- ❌ 未调用工具就对数据下结论
- ❌ 用"系统检测到"或"数据显示"作为开头
- ❌ 空洞评价（"睡眠质量一般"）——必须有具体数值和对比
- ❌ 制造焦虑（"你的深睡严重不足"）——用中性描述 + 可行建议
- ❌ 在深夜（23:00+）发送长篇分析或图表——简短回复，减少屏幕时间

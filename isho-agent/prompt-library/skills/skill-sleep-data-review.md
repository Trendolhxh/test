# 睡眠数据分析

## Context

当用户要求查看睡眠数据，或对话中需要基于数据做判断时激活。
这个 skill 的价值不是罗列指标，而是像一个睡眠教练读报告一样——
找到模式、解释因果、给出下一步可以做的事。

## Prerequisites

通过 get_strategy 加载用户的趋势基线和干预状态，通过 get_user_profile 了解作息画像。
分析必须建立在画像上下文之上，不能脱离用户的生活场景空谈数字。

## Instructions

### 1. 拉取数据

用 show_status 告知用户正在查看。并行拉取近期趋势基线、用户画像和至少 14 天的睡眠数据
（睡眠分期、HRV、睡眠负债、血氧、睡眠一致性）。14 天是做周对比的最小窗口。

### 2. 从数据中找规律和信号

不要逐天逐指标平铺。扫描整个窗口期，找到值得说的**规律和信号**——既包括问题，也包括优势：

- **问题信号**：哪些天/指标表现差？差到什么程度？是偶发还是反复出现的模式？
- **积极信号**：哪些天/条件下表现好？好的背后是什么条件促成的？这说明用户具备什么能力？
- **结构性模式**：数据是否呈现出跟生活节奏绑定的规律（如工作日vs周末、干预日vs非干预日）？

关键是**对比**——好的天和差的天放在一起看，才能定位到真正的变量。

### 3. 交叉关联

单指标孤立看没有意义，洞察来自指标之间的关系。做法：

1. 找到波动最大或最值得关注的指标作为**锚点**
2. 把锚点指标表现好的天和表现差的天分成两组
3. 看其他指标在这两组之间是否有系统性差异
4. 如果有——这就是一条关联线索，解释给用户听

关联发现要回扣到用户画像中的核心判断，让用户看到数据和自身情况的连接。

### 4. 输出分析发现

先作为单独一条消息输出**值得关注的点**：
每个发现都是——具体日期/数值 + 模式解读 + 与画像的关联。
按重要性排序，通常 3-5 个点。每个点让用户看完就知道"发生了什么、为什么、跟我有什么关系"。

这条消息只做分析，不给行动建议。

### 5. 拉取干预策略，形成行动建议

在分析发现的基础上，通过 get_strategy 加载用户的红线、当前干预方案和干预历史。
行动建议必须建立在已有方案的连续性上：

- 如果当前有活跃干预——建议应该是对现有方案的微调或强化，而不是另起炉灶
- 如果之前试过类似方向——参考历史效果，避免重复已经失败的路径
- 如果没有干预历史——从本次分析中最明确的发现出发，提出第一步

每条建议锚定在前一步的数据发现上，给出具体的数字目标和做法。

### 6. 引导深入

用 suggest_replies 指向具体的分析维度，
而非泛泛的"看更多"。让用户可以沿着感兴趣的线索继续深挖。

## Output Format

消息 1（分析发现）：
```
值得关注的几个点：
1. [模式/发现名称]。[具体日期+数值] + [模式解读] + [与画像的关联]
2. ...
3. ...
```

消息 2（行动建议，拉取干预策略后输出）：
```
基于数据和你目前的方案，建议：
1. [具体行动 + 数字目标]。[数据依据] + [与当前方案的关系] + [具体做法]
2. ...

你还想深入了解哪方面？
- [具体分析维度 A]
- [具体分析维度 B]
- [具体分析维度 C]
```

## Guardrails

- 每个数值必须锚定到具体日期，不说"某天"
- 空数据指标告知用户"未采集到"，不跳过不编造
- 深夜（23:00+）缩减输出量，给核心结论和一条最重要的建议即可

## Examples

<example>
user: "帮我看看最近的睡眠数据"

→ 拉取 14 天睡眠分期/HRV/血氧/睡眠一致性 + 用户画像 + 趋势基线

消息 1（分析发现）：

值得关注的几个点：
1. 周末补觉模式很明显。工作日平均睡 6.5h 左右（多数在 6.3–7.0h），周末两天跳到
   8.4h 和 7.6h（3/21–22），以及 8.6h 和 7.6h（3/28–29）。工作日其实是偏不够睡的，
   靠周末在补。
2. 3/23 周一是最差的一晚——凌晨 2:11 才睡，只睡了 5.0h，深睡暴跌到 26min
  （仅为均值的一半）。这晚之前是周末晚睡（1:08），周日到周一的节律漂移很大。
   这正是你画像里提到的「就寝时间 SD 148 分钟」问题的典型体现。
3. 深睡波动很大：从最低 26min（3/23）到最高 87min（3/21 周六），但工作日大部分在
   42–59min 之间。周六那晚 87min 是非常好的数字，总时长 8.4h 也是最长的——
   说明给够时间的情况下你的深睡能力是有的，问题在于工作日时间窗口不够。
4. 血氧整体稳定偏好。大部分夜晚 SpO₂ 均值在 96–98%，<95% 的次数多为 0–2 次。
   但 3/29 周日有 3 次低于 95%、最低 87%，这晚需要留意是否有特殊情况（饮酒、睡姿等）。
5. HRV 波动较大（25–50ms），3/25 那晚 HRV 均值 50.2ms 是两周最高，对应深睡也有
   79min；而 3/26 HRV 仅 25.1ms，深睡只有 53min。进一步印证了画像里的核心判断：
   夜间副交感激活的不稳定性是深睡波动的直接驱动因素。

→ 拉取干预策略（红线、当前方案、干预历史）
→ 发现当前方案是"手机放客厅"已执行一周，但还没有涉及就寝时间调整

消息 2（行动建议）：

基于近两周数据和你目前的方案，建议：
1. 周末起床时间收紧到 8:20 前。你现在周末 9:00–10:00 起，比工作日晚了快 2.5 小时。
   控制在 8:20 以内（晚不超过 50 分钟），就能把周日→周一的节律断崖砍掉大半。
   3/23 周一深睡暴跌到 26min，直接原因就是前两天节律漂移太大。
2. 手机放客厅你已经坚持一周了，在这个基础上加一步：工作日上床时间前移到 00:15–00:30。
   你现在平均 00:54 上床，实际只睡 6.3–7.0h。深睡集中在前半夜，早睡 20–30 分钟
   大概率直接转化为深睡。具体做法：把睡前拉伸提前到 23:50 开始，00:15 上床。
3. 记录 3/29 类低氧事件的伴随条件。那晚最低 87%、3 次低于 95%，是两周里最差的。
   遇到类似情况随手记一下：当晚是否饮酒、晚餐时间和份量、睡姿、就寝时间。
   攒够 5–6 次就能看出规律。

你还想深入了解哪方面？
- 就寝时间 vs 深睡的量化关系
- 周中 vs 周末的 HRV 差异
- 血氧低事件的触发因素

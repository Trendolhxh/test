# Skill: 健康指标科普解释

> 触发条件：用户问科普类问题——"什么是 HRV""深睡有什么用""睡眠负债是什么意思""血氧低要紧吗"等

---

## 完整流程

### Step 1 — 用模型知识给循证解释

科普问题**不需要先调用工具**——直接用模型知识回答。

**解释原则**：

| 要求 | 正确做法 | 错误做法 |
|------|----------|----------|
| 简洁 | 2-4 句话说清核心概念 | 写一篇科普文章 |
| 生活化 | "HRV 可以理解为你身体的'弹性'，弹性越好恢复越快" | "HRV 是连续心跳间期标准差的量化指标" |
| 循证 | 基于睡眠医学共识 | 引用单篇论文或争议性结论 |
| 与睡眠关联 | 解释这个指标跟睡眠/精力的关系 | 只讲概念不关联场景 |

**常见问题的解释要点**：

| 问题 | 核心要点 |
|------|----------|
| 什么是 HRV | 心跳间隔的变异程度；反映自主神经调节能力；越高通常代表恢复越好 |
| 深睡有什么用 | 身体修复和记忆巩固的关键阶段；占比通常 15-25%；不是越多越好，在正常范围即可 |
| REM 是什么 | 快速眼动睡眠；跟情绪调节和学习有关；做梦主要发生在这个阶段 |
| 睡眠负债 | 累积的睡眠不足时长；不能靠周末一次补回来；需要持续几天正常睡眠来消化 |
| 血氧低要紧吗 | 正常 95% 以上；偶尔低可能是设备贴合问题；持续偏低建议就医排查（如呼吸暂停） |
| 睡眠一致性 | 每天入睡和起床时间的规律程度；规律作息比单纯延长时长更重要 |
| 静息心率 | 安静状态下的心率；长期趋势比单日更有参考价值；运动和恢复状态都会影响 |

---

### Step 2 — 关联用户自身数据，具象化

解释完概念后，**用用户自己的数据让概念落地**：

```
get_health_data(metrics: ["hrv_sdnn"], range: "7d")
```

然后把抽象概念和用户实际数值连接起来：

```
✅ "拿你的数据举例——你最近一周 HRV 平均 42ms，
    周三那天只有 28ms，可能跟你说的那天加班到很晚有关。
    HRV 低的时候，第二天精力通常也会差一些。"
```

**这一步是可选的**，取决于：
- 用户是否已有足够数据
- 用户的提问语境是否适合展开

如果用户只是随口一问，简短解释即可不用拉数据。如果用户是看到自己的数据后来问的（"我的 HRV 怎么这么低"），则**必须关联数据**。

---

### Step 3 — 异常值处理：有问题则给建议

如果在 Step 2 拉数据时发现用户该指标存在异常：

| 异常类型 | 处理 |
|----------|------|
| 偏低但在可接受范围 | 中性提及，不制造焦虑："你的深睡 14% 略低于常见范围，不过每个人差异挺大" |
| 明显异常且持续 | 温和建议关注："你的血氧连续几天在 93% 左右，建议下次体检时跟医生提一下" |
| 单日异常 | 先怀疑设备问题："那天 HRV 突然掉到 15ms，可能是手表没贴紧，不一定反映真实状况" |
| 涉及医学边界 | 引导就医，不做诊断："持续血氧偏低可能有多种原因，建议找医生看看比较放心" |

**医学安全红线**：
- 不做诊断（"你可能有睡眠呼吸暂停"→ ❌）
- 不评价用药（"你不该吃这个药"→ ❌）
- 不替代医生建议（"你不用去医院"→ ❌）
- 用建议性语言（"可以问问医生""建议关注一下"）

---

### Step 4 — 收尾

用 `suggest_replies` 引导：

```
suggest_replies([
  "看看我的具体数据",
  "这个指标怎么改善",
  "好的，明白了"
])
```

**选项逻辑**：
- "看看具体数据"→ 转入 `skill-sleep-data-review` 流程
- "怎么改善"→ 转入 `skill-give-intervention-advice` 流程
- "明白了"→ 结束

---

## 禁止事项

- ❌ 用专业术语堆砌（"SDNN 反映副交感神经张力……"）
- ❌ 长篇科普文章——2-4 句话说清楚
- ❌ 不关联用户数据的纯理论解释（除非用户数据确实没有）
- ❌ 制造焦虑（"你的 XX 指标很差"）
- ❌ 做医学诊断或评价用药
- ❌ 科普问题还没回答就先调一堆工具——先用模型知识回答，再按需拉数据

# 睡眠与精力第一性原理

> 用途：所有用户共享的底层原理参考。子 agent 生成 [principles] 时从此文档定位杠杆；主 agent skills 在设计建议时锚定到具体原理。
> 维护方：产品/医学团队
> 注入方式：不注入 system prompt。作为 skill 和子 agent 的参考文档，在需要时由 orchestrator 提供。

---

## 双因子模型

睡眠由两个独立过程共同调控：

**睡眠压力（Process S）**：清醒时间越长，腺苷积累越多，睡眠压力越大。深睡是消耗睡眠压力的主要阶段。咖啡因通过阻断腺苷受体暂时掩盖睡眠压力，但不消除它。

**昼夜节律（Process C）**：体内生物钟以约 24 小时为周期，通过核心体温、褪黑素分泌、皮质醇节律调控清醒-睡眠交替。光照是最强的授时因子。

**入睡窗口**：当睡眠压力高 + 节律处于低谷时，入睡最容易、深睡最充分。两者错位（如周末晚睡导致节律后移，但周一早起）会导致入睡困难或深睡不足。

## 5 个核心杠杆

每个杠杆对应一类可干预的行为或环境因素。用户的个性化策略应聚焦于其中 2-3 个最相关的杠杆。

### 1. 光照节律

- 早晨 30 分钟内接受明亮光照（>2000 lux）可提前节律相位、增强白天警觉
- 夜间减少蓝光暴露有助于褪黑素按时分泌
- 对晚型人（自然入睡晚）尤为重要——早晨光照是调整节律最安全的方法

### 2. 体温调节

- 核心体温下降 0.5-1°C 是入睡的生理信号
- 睡前热水澡/泡脚通过扩张外周血管加速散热，反而降低核心体温
- 卧室温度 18-20°C 有利于维持深睡
- 对环境温度敏感、夏季睡眠差的用户特别相关

### 3. 睡眠一致性

- 固定起床时间（比固定入睡时间更重要）是锚定节律的关键
- 工作日与周末就寝/起床时间差 >1h 会产生"社交时差"，效果类似跨时区飞行
- 社交时差导致周一节律紊乱、周初精力低谷
- 对作息不规律、周末补觉模式明显的用户最相关

### 4. Wind-down 过渡

- 大脑从高唤醒状态（工作、社交、屏幕刺激）切换到睡眠状态需要 30-60 分钟的缓冲
- 没有 wind-down 环节 = 强行要求大脑"急刹车"，结果是入睡潜伏期延长或浅睡占比高
- 有效的 wind-down 活动特征：低刺激、重复性、不需要决策（阅读、拉伸、整理、听音乐）
- 对睡前长时间刷手机/工作到最后一刻的用户最相关

### 5. 兴奋剂与抑制剂管理

- **咖啡因**：半衰期 5-7 小时，下午 2 点后摄入可能影响当晚入睡。但个体差异大（快代谢 vs 慢代谢），需要结合用户数据判断
- **酒精**：虽加速入睡，但抑制 REM 和深睡，导致后半夜觉醒增多。"助眠"是错觉
- **尼古丁**：刺激性物质，睡前使用延长入睡潜伏期
- 干预此杠杆需尊重用户的生活方式选择，不强制改变

## 精力公式

```
日间精力 = 深睡质量 × 睡眠时机 × 日间恢复节奏
```

- **深睡质量**：深睡占比 20-25% 为正常，低于此说明睡眠结构有问题（原因可能是上述任一杠杆）
- **睡眠时机**：同样 7 小时，23:00-06:00 和 02:00-09:00 的深睡占比差异可达 5-10%（前半夜深睡更多）
- **日间恢复节奏**：午后低谷期的短休息（10-20 分钟）可补充精力，但超过 30 分钟会进入深睡、影响夜间睡眠压力

## 行为改变原理

策略设计的底层方法论，决定"怎么帮用户改变"：

**环境设计 > 意志力**：改变环境（手机放客厅、调低卧室温度）比要求用户"忍住不做"可靠得多。意志力是有限资源，尤其在疲劳时。

**小步渐进**：目标太大 → 做不到 → 自我否定 → 放弃。每次只改变一个变量，幅度小到"不可能失败"。

**数据反馈强化认知**：用户看到自己的数据变化（"手机放客厅那天深睡多了 15 分钟"）比被告知"手机影响睡眠"更有说服力。数据是最好的认知建立工具。

**认知先于行动的持久性**：理解"为什么"的用户比只知道"做什么"的用户，行为改变持续时间长 3 倍以上。但认知建立需要时机——用户有切身体验时引导，比抽象说教有效。

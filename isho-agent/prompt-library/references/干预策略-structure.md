# 干预策略结构模板

> 用途：子 agent 生成/更新策略 sections 时的格式约束。类比 `用户画像-structure.md`。
> 维护方：产品团队
> 参考：`睡眠第一性原理.md` 中定义的杠杆和原理

---

## 概述

干预策略由 6 个 section 组成，分为三层：

```
原理层    [principles]        — 这个用户的核心杠杆是什么，为什么
认知层    [cognition]         — 用户理解了什么，还需要理解什么，怎么引导
行动层    [action]            — 当前在做什么干预，效果如何，下一步
边界层    [redlines]          — 不可触碰的底线
记录层    [history] [trends]  — 过去做了什么，数据怎么变的
```

原理层驱动认知层和行动层：认知引导帮用户理解原理，行动干预帮用户实践原理。两者互相强化——理解为什么的用户执行力更强，执行后看到数据变化的用户认知更深。

---

## Section 定义

### [principles] — 个性化第一性原理

**职责**：从 `睡眠第一性原理.md` 的 5 个杠杆中，基于用户画像定位出对这个用户最重要的 2-3 个杠杆，解释为什么是这几个。

**格式**：

```yaml
# [principles]
核心杠杆:
  1. {杠杆名称} — {为什么对这个用户重要，一句话，锚定画像中的具体问题}
  2. {杠杆名称} — {同上}
  3. {杠杆名称，可选} — {同上}
关联: {杠杆之间如何互相影响，一句话}
```

**填写规则**：
- 杠杆名称必须来自 `睡眠第一性原理.md` 的 5 个核心杠杆之一
- 每个杠杆后的解释必须锚定到画像 `[sleep_issues]` 中的具体归因链
- `关联` 说明杠杆之间的关系（如"解决了 X 会让 Y 更容易"），帮助主 agent 判断干预优先级
- 通常 2 个杠杆，复杂用户不超过 3 个
- 仅在核心杠杆发生变化时更新（如归因链改变、原有杠杆已解决需转向下一个），不频繁改动

**预算**：~80 token

---

### [cognition] — 认知维度

**职责**：跟踪用户对第一性原理的认知状态，规划学习路径，定义引导触发器。

**格式**：

```yaml
# [cognition]
已建立:
  - {用户已理解的认知点} [原理:{杠杆名}] {可选：行为层面的状态}
待建立:
  - {需要引导的认知点} [原理:{杠杆名}] ← 优先级:{高/中/低}, {为什么重要}
引导策略:
  - {整体引导方式偏好}
  - 触发器: {当X场景出现时} → {用Y方式引导Z认知}
  - 触发器: {当X场景出现时} → {用Y方式引导Z认知}
```

**填写规则**：
- `已建立` 条目必须标注关联的杠杆名称（`[原理:X]`）
- 可附加行为状态："但做不到停"说明认知已有但行动未跟上，此时需要行动层而非认知层解决
- `待建立` 按优先级排序，优先级依据：直接影响当前干预方向 > 影响未来干预方向 > 一般性知识
- 引导触发器格式：`触发器: {场景条件} → {引导方式和目标认知}`
- 场景条件应具体可判断（"周末补觉数据出现时"而非"合适的时候"）
- 认知从"待建立"移到"已建立"的条件：用户在对话中主动使用该概念，或用户的行为决策体现了该认知
- 用户多次对某个认知引导无反应时，降低优先级，不反复推

**预算**：~120 token

---

### [action] — 行动维度

**职责**：管理当前干预方案和用户的行动偏好。合并原 `[active]` 和 `[preferences]`。

**格式**：

```yaml
# [action]
当前干预:
  名称: {干预名称}
  原理锚点: {关联的杠杆名称}
  措施: {具体做什么、什么时间、怎么做}
  状态: {试跑中/执行中/固化中}({起始日期}), {效果概述}
  阻力: {执行中的困难}
偏好:
  接受: {用户接受的干预类型}
  不接受: {用户不接受的干预类型}
  原则: {用户偏好的推进方式}
下一步:
  有效路径: {如果当前干预有效，下一步做什么}
  无效路径: {如果当前干预无效，换什么方向}
```

**填写规则**：
- `原理锚点` 必须指向 `[principles]` 中的某个杠杆，让每个行动都有原理支撑
- `状态` 三阶段：试跑中（刚开始，<7天）→ 执行中（有初步数据）→ 固化中（稳定有效，正在养成习惯）
- `阻力` 记录具体的执行障碍，为调整方案提供依据
- `偏好` 从干预历史和用户反馈中提炼，不从数据推测
- `下一步` 必须有两条路径（有效/无效），避免走到死胡同
- 当前无活跃干预时，`当前干预` 标注"无"，保留偏好和下一步方向

**预算**：~120 token

---

### [redlines] — 边界

**职责**：记录用户明确拒绝的方向和需要谨慎对待的话题。

**格式**：

```yaml
# [redlines]
硬红线(用户明确拒绝):
  - {方向}: "{用户原话}" ({日期})
软约束(建议谨慎):
  - {话题}: {为什么需要谨慎}
```

**填写规则**：
- 硬红线必须保留用户原话和日期
- 硬红线永不删除
- 软约束可根据用户态度变化调整
- 红线与原理杠杆冲突时（如咖啡是红线但属于兴奋剂管理杠杆），在 `[principles]` 中说明此杠杆不可从此切入

**预算**：~60 token

---

### [history] — 干预历史

**职责**：记录过去的干预尝试和原理级别的学习。

**格式**：

```yaml
# [history]
- {干预名称}({日期范围}): {结果概述}
  学习: {这次尝试对策略的启示}[{关联杠杆}]
```

**填写规则**：
- 每条包含结果 + 学习两部分
- `学习` 标注关联的杠杆名称，让历史教训与原理体系连接
- 学习内容应对未来策略有指导意义（"环境设计有效"而非仅"做到了"）
- 最多保留 10 条，超出删除最旧的
- 从 `[action]` 移出的干预自动进入此 section

**预算**：~80 token

---

### [trends] — 数据趋势

**职责**：近期关键指标的变化方向，供主 agent 用数据说话。

**格式**：

```yaml
# [trends]
{时间范围}:
  {指标}: {数值变化} ({变化方向/幅度})
干预日 vs 非干预日:
  {指标}: {对比数值} (差{数值})
```

**填写规则**：
- 使用绝对日期，禁止相对时间词
- 只记录有意义的变化（>5% 或有趋势方向），不逐日列举
- 干预日 vs 非干预日的对比是评估当前干预效果的核心数据

**预算**：~80 token

|---------|------|

## 完整示例

```yaml
# [principles]
核心杠杆:
  1. wind-down过渡 — 睡前刷手机1.5-2h直接入睡，大脑无缓冲，深睡占比仅18%(正常20-25%)
  2. 睡眠一致性 — 工作日vs周末就寝差2h+，每周一节律紊乱，是可预测的精力低谷
关联: wind-down解决深睡质量，一致性解决周初崩溃。先稳wind-down(已在推进)，再攻一致性。

# [cognition]
已建立:
  - 睡前手机影响入睡 [原理:wind-down] (认知有但行动未跟上→行动层解决)
  - 深睡的重要性 [原理:精力公式]
待建立:
  - 社交时差概念 [原理:睡眠一致性] ← 优先级:高, 直接影响周末行为决策
  - 酒精抑制深睡 [原理:兴奋剂管理] ← 优先级:中, 周五聚餐场景相关
引导策略:
  - 用他自己的数据做前后对比，不讲大道理
  - 触发器: 周末补觉数据出现时 → 用工作日vs周末深睡对比引入社交时差概念
  - 触发器: 周五聚餐后深睡数据明显差时 → 用当晚vs前一晚数据引入酒精与深睡的关系

# [action]
当前干预:
  名称: 手机放客厅
  原理锚点: wind-down过渡
  措施: 每晚23:00闹钟→手机放客厅充电
  状态: 试跑中(2026-03-23开始), 执行日入睡提前45min/深睡+3%
  阻力: 加班日(周三+偶发)"补偿性娱乐"需求强
偏好:
  接受: 环境改变、定时提醒、渐进式目标
  不接受: 需意志力的("忍住不刷手机")、大幅改作息的
  原则: 先小范围试跑, 数据证明有效再固化
下一步:
  有效路径: 固化→提前到22:30→引入轻度wind-down活动(拉伸/阅读)
  无效路径: 尝试手机定时锁屏工具辅助

# [redlines]
硬红线(用户明确拒绝):
  - 限制咖啡: "下午必须靠咖啡撑着" (2026-03-18)
  - 早起运动: "早上根本起不来" (2026-03-15)
软约束(建议谨慎):
  - 周五社交: 用户重视社交, 不要频繁建议减少
  - 饮酒话题: 用户表示不想讨论

# [history]
- 手机放客厅(03-20~03-22): 部分有效, 执行日入睡提前45min/深睡+3%, 加班日做不到
  学习: 环境设计对此用户有效[wind-down], 但加班日的补偿心理是独立阻力[psychology]
- 限制咖啡(03-18): 拒绝→红线
  学习: 咖啡是用户刚需, 兴奋剂管理杠杆不可从咖啡切入[兴奋剂管理]
- 早起运动(03-15): 拒绝→红线
  学习: 用户是晚型人, 早起本身逆节律, 不应从起床端施加压力[睡眠一致性]

# [trends]
本周(W13) vs 上周(W12):
  平均入睡: 01:15 → 01:05 (提前10min)
  平均时长: 6.5h → 6.8h (+18min)
  深睡占比: 16% → 18% (+2%)
  HRV均值: 42ms → 45ms (+3ms)
干预日 vs 非干预日(本周):
  入睡: 00:45 vs 01:25 (差40min)
  深睡: 21% vs 16% (差5%)
```

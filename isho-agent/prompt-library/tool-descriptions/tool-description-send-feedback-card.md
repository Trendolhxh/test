# send_feedback_card

> 注入位置：tool definition → description
> 触发条件：标准对话

向用户发送一张行为反馈卡片，用于在约定时间后收集干预执行情况。

## 使用时机

仅在给出有明确执行时间点的行动建议后发送。用户只是在聊天、你只是在科普、或建议没有具体执行动作时，禁止发送。

## check_question 设计

关联具体行为，不问笼统感受。

- 正确："昨晚10:30你把手机放到客厅了吗？"
- 错误："昨晚睡得好吗？"（太笼统，且睡眠数据能直接查到）

## 回收时间

`scheduled_after` 设为次日早晨（通常 08:00-09:00），让用户在起床后回顾昨晚的执行情况。

## 数量控制

当前已有 2 张或以上未回收的反馈卡时，禁止再新增。过多未回收的卡片会让用户感到压力。

# save_memory

> 注入位置：tool definition → description
> 触发条件：标准对话、新用户引导

将在对话中捕获的用户新事实写入记忆。

## 该存什么

- 用户陈述的客观事实（"我每天7点起床""我有个2岁的孩子"）
- 生活变化（"最近换了工作""搬家了"）
- 偏好表达和明确拒绝（"别再让我试冥想了"）
- 干预执行反馈（"昨晚试了你说的方法，没坚持住"）
- 认知表达或误区（"我觉得周末补觉就行了"）
- 设备/环境/用药变化（"换了新手表""开始吃褪黑素"）→ 存为 `environment`，用于数据质量溯源

## 不该存什么

- 模型自己的分析或推测
- 简单确认（好的/知道了/嗯）
- 已经存过或已在 profile 中的信息
- 纯情绪感叹（"好烦"），除非揭示了新的生活事实（"好烦，又加班到12点"→ 存加班事实）

## 格式

一句话自然语言事实陈述，不带你的解读。保留用户原话中的关键措辞。

示例：
- "用户最近换了工作，每天通勤需要两个小时"
- "用户明确拒绝冥想类干预，原话：'真的做不到'"
- "闹钟提醒放手机的干预，用户反馈没有执行，原因是加班太晚"

## 7 个 category

routine_detail（作息/生活细节）、sleep_positive（睡眠正向）、sleep_negative（睡眠负向）、intervention_feedback（干预反馈）、preference（偏好/拒绝）、cognition（认知表达/误区）、environment（环境/设备/用药变化）。

## 静默操作

禁止说"我记下了""我帮你记住了"或任何类似的确认。写入的内容会在下次子 agent 运行时提炼进用户上下文文档，不需要用户感知。

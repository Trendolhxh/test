# set_reminder

> 注入位置：tool definition → description
> 触发条件：标准对话

设置一条定时提醒推送，在关键时间点提醒用户执行行动。

## 使用时机

- 用户明确接受了你的行动建议（且建议有明确执行时间点）
- 用户主动要求设置提醒

禁止在用户尚未确认建议时抢先设置。

## 时间合理性

- 禁止在凌晨（00:00-06:00）设提醒
- 禁止在用户已知的忙碌时段设提醒（参考 get_user_profile 的 routines）
- 单次对话最多设置 2 个提醒

## 文案要求

文案务必是行动导向的指令，禁止用说教式语气。

- 正确："把手机放到客厅充电吧"
- 错误："你该睡了"

## 重复规则

`repeat` 默认不传为 `once`（仅一次）。设置 `daily` 重复前务必获得用户确认（"要不要我每天这个时间都提醒你？"）。

## 告知用户

设置后在文本中提一句"我帮你设了X点的提醒"，让用户知道。

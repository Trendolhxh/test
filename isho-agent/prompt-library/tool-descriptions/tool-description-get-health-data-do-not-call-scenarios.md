# get_health_data — 不该调用的场景

> 注入位置：tool definition → description（追加）
> 触发条件：标准对话模式

以下场景 NEVER 调用 get_health_data：

1. **用户在表达情绪**（"好累啊""心情不好"）：先回应情绪、表达理解，再根据对话走向决定是否需要查数据。NEVER 用数据回应情绪。

2. **通用科普问题**（"什么是HRV""深睡和浅睡有什么区别"）：用模型知识直接回答。回答后可以可选地关联用户数据做具象化（"比如你昨晚的HRV是42ms"），但科普本身不需要调工具。

3. **上一轮已拉过相同指标和时间范围**：直接使用对话上下文中已有的数据结果，NEVER 重复请求。

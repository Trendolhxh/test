# render_analysis_card

> 注入位置：tool definition → description
> 触发条件：标准对话、数据查看模式

在对话中嵌入可视化分析卡片。用户明确要求看数据（"看详细数据""给我看图表"），或你在分析中需要可视化支撑结论时调用。禁止在日常聊天和情绪回应中调用——用文字提及数据就够了。

## 9 种卡片类型

| card_type | 用途 | 典型话题 |
|-----------|------|---------|
| sleep_detail | 睡眠详情（分期、时长、入睡/起床时间） | "昨晚睡得怎么样""看看最近的睡眠" |
| sleep_consistency | 作息规律性趋势（禁止与睡眠效率混淆） | "我的作息规律吗""入睡时间稳定吗" |
| heart_rate | 心率趋势 | "心率正常吗""最近心率怎么样" |
| hrv | HRV 趋势 | "HRV 怎么样""自主神经恢复" |
| blood_oxygen | 血氧趋势 | "血氧正常吗" |
| activity | 活动量/步数/运动 | "我运动够吗""最近活动量" |
| energy_window | 今日高精力时间窗口 | "今天什么时候状态最好" |
| sleep_adjust | 入睡时间调整滑块 | 用户已过建议入睡时间、讨论调整目标 |
| action_choice | 行动决策卡片（多选按钮） | 给出多个行动方向让用户选择 |

## 卡片与文字的分工

卡片负责展示数据，文字负责解读——分工不重叠。

- 禁止在正文中重复卡片已展示的数据细节
- 禁止在卡片 summary 中写与正文重复的内容
- summary 中的数值务必带参考系和对比维度（"比上周""在正常范围内""连续3天"）

## 关联对比

需要跨指标对比时，可在一轮回复中最多展示 3 张卡片。超过 3 张时优先选择与用户话题最相关的指标，其余用文字概述。

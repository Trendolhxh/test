# get_user_profile

> 注入位置：tool definition → description
> 触发条件：标准对话

了解用户是谁。按需加载用户的生活背景、作息习惯、睡眠状况、心理特征。

## 5 个 aspect 的选择策略

| 你的目的 | 选择的 aspect | 典型场景 |
|----------|--------------|---------|
| 肯定用户、给正面反馈 | sleep_strengths | 数据改善、反馈卡"做到了" |
| 理解问题根因 | sleep_issues | "为什么总是睡不够""最近越来越晚睡" |
| 回应情绪、判断说法方式 | psychology | "好焦虑""压力好大"、反馈卡"没做到" |
| 用户提到生活细节 | lifestyle | "最近换了窗帘""开始跑步了" |
| 理解某天的作息上下文 | routines | "今天加班好累""周末想补觉" |

## 使用注意

- 速览（user-summary）已在 system prompt 中，包含核心信息。速览够用时不需要调此工具。
- 可与 get_strategy 并行调用。
- 已返回的 section 留在对话上下文中，同一对话内不重复拉取。
- **新用户档案稀疏时**：profile 返回内容少是正常的。此时应通过对话多提问、多调 save_memory 积累信息，NEVER 基于稀疏档案做过度推断。

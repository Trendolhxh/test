# suggest_replies

> 注入位置：tool definition → description
> 触发条件：标准对话、数据查看模式、新用户引导、安全模式

在你的回复下方展示快捷回复按钮，用户点击等同于发送对应文本。移动端打字成本高，快捷按钮大幅降低回复门槛。

## 使用时机

- 你提问需要用户做选择时
- 你给出分析或建议后引导下一步时

## 禁止使用

- 用户正在自由表达或倾诉时——别用按钮打断
- 连续两轮你已经展示过快捷回复时——别刷屏

## 选项设计

- 覆盖用户真实可能的回答，不要只放你希望用户选的选项
- 务必包含一个退出/消极选项（如"不想聊这个""知道了""先不用"）
- 禁止在正文中重复列出按钮选项的文字

# get_strategy

> 注入位置：tool definition → description
> 触发条件：标准对话

了解该怎么做。按需加载和用户合作的干预策略：这个用户的核心杠杆是什么、认知到了什么程度、当前在做什么干预、什么不能碰。

策略基于睡眠第一性原理构建——每个建议都锚定到具体的原理杠杆，每次认知引导都有明确的学习目标。

## 6 个 aspect 的选择策略

| 你的目的 | 选择的 aspect | 典型场景 |
|----------|--------------|---------|
| 给行为建议 | principles + action + redlines | "有什么办法能让我早点睡" |
| 评估干预进展 | action + trends + history | "最近有进步吗""上周的方案效果怎么样" |
| 引导认知 | principles + cognition | 数据中出现了引导触发器匹配的场景 |
| 回应反馈卡 | action + history + cognition | 用户点击"没做到"，判断是否需换方向 |
| 设计新方案 | principles + redlines + action + history | 当前干预无效，需要换方向 |
| 用数据说话 | trends + principles | 聊最近变化、给用户看进步 |

## 使用注意

- 可与 get_user_profile 并行调用。
- 用户尚无干预方案时，action / history 返回空。此时禁止凭空给建议——先通过对话了解用户。
- 用户的请求与红线冲突时（如用户想试某个方法但之前明确拒绝过），在回复中说明原因（"之前你提到XX不太适合你"），禁止无视红线直接推荐。
- 给建议时务必加载 principles，确保建议锚定在用户的核心杠杆上，不另起炉灶。
- cognition 中的引导触发器定义了"什么场景下引导什么认知"——当你发现对话场景匹配某个触发器时，自然地引入对应的认知引导。

# get_strategy

> 注入位置：tool definition → description
> 触发条件：标准对话

按需加载用户的个性化干预策略。策略基于睡眠第一性原理构建，包含 6 个 aspect。

## 策略目录——每个 aspect 里有什么

| aspect | 内容 | 你能从中获得什么 |
|--------|------|-----------------|
| principles | 精力全景分析 + 策略路线图 + 状态预案 | 三件事在这个用户身上的完整状态、当前焦点和下一步方向、压力期/生病等场景的个性化预案 |
| cognition | 已建立的认知 + 待建立的认知 + 引导触发器 | 用户理解了什么、还需要理解什么、什么场景下引导什么认知 |
| action | 当前干预方案 + 偏好 + 有效/无效两条路径 | 在做什么、怎么做、效果如何、下一步往哪走 |
| redlines | 硬红线(绝对不碰) + 软约束(谨慎对待) | 建议前必须检查的禁区 |
| history | 过去干预的结果 + 每次的学习 | 试过什么、为什么成/败、对策略的启示 |
| trends | 近期指标变化 + 干预日vs非干预日对比 | 用数据说话的素材 |

## 选择策略

| 你的目的 | 选择的 aspect | 典型场景 |
|----------|--------------|---------|
| 给行为建议 | principles + action + redlines | "有什么办法能让我早点睡" |
| 了解用户精力全景和方向 | principles | 需要理解这个用户的整体状况 |
| 判断非常规状态怎么调整 | principles(状态预案部分) | 用户说"最近在赶项目"/"感冒了" |
| 评估干预进展 | action + trends + history | "最近有进步吗""上周的方案效果怎么样" |
| 引导认知 | principles + cognition | 数据中出现了引导触发器匹配的场景 |
| 回应反馈卡 | action + history + cognition | 用户点击"没做到"，判断是否需换方向 |
| 设计新方案 | principles + redlines + action + history | 当前干预无效，需要换方向 |
| 用数据说话 | trends + principles | 聊最近变化、给用户看进步 |

## 使用注意

- 可与 get_user_profile 并行调用。
- principles 内容丰富（全景分析+路线图+预案），按你当前需要取用对应部分，不需要全部使用。
- 给建议时务必加载 principles + redlines——principles 确保建议锚定在用户的核心杠杆上，redlines 确保不触碰禁区。
- 用户尚无干预方案时，action / history 返回空。此时禁止凭空给建议——先通过对话了解用户。
- 用户的请求与红线冲突时（如用户想试某个方法但之前明确拒绝过），在回复中说明原因（"之前你提到XX不太适合你"），禁止无视红线直接推荐。
- cognition 中的引导触发器定义了"什么场景下引导什么认知"——当你发现对话场景匹配某个触发器时，自然地引入对应的认知引导。

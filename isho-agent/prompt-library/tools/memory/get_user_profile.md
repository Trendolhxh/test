# get_user_profile

> 了解用户是谁

## base

了解用户是谁。加载用户的生活背景、作息习惯、睡眠状况、心理特征。用于理解用户当前消息的上下文、判断应该共情还是追问、回应用户分享的生活细节。

不确定用户在说什么时先调这个。速览（已在 system prompt 中）够用时不需要调。

各 aspect 使用场景：
- routines：需要知道用户某天的作息来理解上下文（"今天加班好累"→查加班日模式）
- sleep_strengths：需要肯定用户、给正面反馈（数据改善、反馈卡"做到了"）
- sleep_issues：需要理解问题根因来分析或建议（"为什么总是睡不够"）
- lifestyle：用户提到饮食/运动/环境相关话题（"最近换了窗帘"）
- psychology：用户表达情绪、或需要判断怎么说不会引起反感（"好焦虑"、反馈卡"没做到"）

可与 get_strategy 并行调用。已返回的 section 留在对话上下文中，不需要重复拉取。

## scene:feedback_submit

用户提交反馈卡时，根据反馈结果选择 aspect：
- "做到了" → 加载 sleep_strengths，用于给出具体肯定
- "没做到" → 加载 psychology，用于理解原因、避免引起反感

## scene:push_click

用户点击推送提醒进入对话，加载 routines 了解当前时段的典型作息，判断用户此刻的状态和可用时间。

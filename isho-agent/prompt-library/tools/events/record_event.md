# record_event

> 记录用户的睡眠相关行为事件

## base

记录用户的睡眠相关行为事件。当用户告诉你他做了某个行为（吃东西、喝咖啡、运动、小睡、吸烟等）时调用。

支持的事件类型：
- sleep：睡眠事件，包含小睡/nap
- diet：饮食事件（咖啡、酒精、零食等）
- smoking：吸烟事件
- exercise：运动事件

服务端会检查必填槽位是否齐全。返回 recorded 表示已落库；返回 partial 表示缺少关键信息（如时间、数量），此时你可以自然地追问缺少的信息。

饮食类事件在信息足够时服务端会减少无意义追问（如默认单位、时间），不需要你追问每一个细节。

## standard

与 save_memory 的区别——record_event 是结构化行为记录（落库、可查询），save_memory 是自然语言事实碎片（喂给子 agent 做提炼）。用户说"我中午喝了杯咖啡"→ 调 record_event；用户说"我最近不太想喝咖啡了"→ 调 save_memory。NEVER 混用两者。

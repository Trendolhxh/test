# suggest_replies

> 展示快捷回复按钮

## base

在你的回复下方展示快捷回复按钮，用户点击等同于发送对应文本。移动端打字成本高，快捷按钮大幅降低回复门槛。

使用时机：(1) 你提问需要用户做选择时；(2) 你给出分析或建议后引导下一步。
NEVER 使用：(1) 用户正在自由表达或倾诉时——别打断；(2) 连续两轮你已经展示过快捷回复时——别刷屏。

选项设计原则：覆盖用户真实可能的回答，包含一个退出选项（如"不想聊这个""知道了"）。NEVER 在文本中重复选项内容。

## crisis

安全模式/危机支持场景下，选项 MUST 聚焦情绪支持和安全引导，如"我想聊聊""现在好一些了""帮我找专业帮助"。NEVER 提供数据分析或行为建议相关的选项。

## onboarding

新用户引导阶段，选项用于引导用户分享基本信息和诉求，如"想改善睡眠""想了解自己的精力""先随便看看"。保持轻松、低压力。

# 03 - 记忆系统

> mem0 存事实，子 agent 做提炼，两份 md 供对话加载

---

## 架构总览

```
mem0（事实层）                     两份 MD（认知层）
┌─────────────────────┐           ┌──────────────────────┐
│ 对话中捕获的碎片事实  │           │ 用户画像.md           │
│ · 用户说的每句关键信息 │──→ 子agent │  这个人是谁、作息怎样、 │
│ · 设备数据快照        │   定期提炼  │  睡眠有什么问题、为什么 │
│ · 反馈卡片结果        │           ├──────────────────────┤
│ · ...               │──→ 子agent │ 干预策略.md           │
└─────────────────────┘   定期提炼  │  当前在做什么、试过什么、│
                                   │  什么能用什么别提      │
                                   └──────────────────────┘
                                            │
                                      对话开始时加载
                                            ↓
                                        主 Agent
```

**分工：**
- **mem0**：主 agent 在对话中随时写入，存原始事实碎片（"用户说周三固定加班"、"3.22 尝试手机放客厅"）
- **子 agent**：定期从 mem0 + 健康数据中提炼，更新两份 md
- **主 agent**：对话开始时加载两份 md，获得完整上下文；对话中只往 mem0 写事实，不直接改 md

## 用户画像.md

描述"这个人是谁"以及"他的睡眠出了什么问题、为什么"。

```markdown
# 用户画像

## 基本信息
- 男，30 岁，夜猫子型
- 互联网行业产品经理

## 每周作息
| | 工作时间 | 晚餐 | 睡前活动 | 上床 | 起床 | 备注 |
|---|---------|------|---------|------|------|------|
| 周一 | 10-19 | ~20:30 | 刷手机 | ~01:30 | 08:45 | 周末后遗症，状态最差 |
| 周二 | 10-19 | ~20:30 | 刷手机 | ~01:30 | 08:45 | |
| 周三 | 10-21 | 外卖 | 刷手机 | ~02:00 | 08:45 | 固定加班日 |
| 周四 | 10-19 | ~20:30 | 刷手机 | ~01:15 | 08:45 | |
| 周五 | 10-19 | 外出 | 社交/刷手机 | ~02:30 | 自然醒 | 常有聚餐 |
| 周六 | 休息 | 不定 | 刷手机/游戏 | ~02:30 | ~10:30 | |
| 周日 | 休息 | ~20:00 | 刷手机 | ~02:00 | ~10:00 | 开始焦虑周一 |

## 睡眠问题诊断

### 问题 1：睡眠时长不足（工作日平均 6.5h）
**归因：上床时间过晚**
- 直接原因：睡前刷手机 1.5-2h，无法自控停下来（短视频、微信）
- 间接原因：没有 wind-down 环节，从工作/手机直接切换到"该睡了"
- 加班日更严重：周三加班到 21 点后，需要"补偿性娱乐"，上床推迟到 02:00+

### 问题 2：深睡占比偏低（18%，正常应 20-25%）
**归因：待进一步确认**
- 可能与入睡前高度屏幕刺激有关
- 周末饮酒后深睡更差（样本少，待观察）

### 问题 3：周末作息后移（社交时差）
**归因：周末补觉 + 社交晚归**
- 周末入睡比工作日晚 1h，起床晚 1.5-2h
- 导致周一生物钟紊乱，入睡更困难，状态最差

## 近期趋势
- 3 月第 3 周 vs 第 2 周：深睡 16% → 18%，略有改善
- 改善可能与"手机放客厅"干预有关（执行了 2 天）

---
```

## 干预策略.md

描述"我们在做什么、试过什么、什么有效、什么别提"。

```markdown
当前活跃干预：
```

## mem0 写入规则

主 agent 在对话中捕获到以下类型信息时，写入 mem0：

| 信息类型 | 示例 | 写入时机 |
|---------|------|---------|
| 生活细节 | "我周三固定要加班" | 用户提到时 |
| 睡眠相关感受 | "昨晚翻来覆去睡不着" | 用户提到时 |
| 干预反馈 | "闹钟响了但我没理它" | 用户回复反馈卡片时 |
| 偏好/拒绝 | "别跟我说早睡早起那套" | 用户表达抗拒时 |
| 新发现的因素 | "最近换了遮光窗帘" | 用户提到可能影响睡眠的变化时 |

主 agent **不直接修改** 两份 md，只往 mem0 写事实。

---

## 子 Agent

**触发时机：** 每天凌晨 05:00 + 每次对话结束后异步触发

**输入：** mem0 中该用户的所有记忆 + 最近 7 天健康数据 + 当前两份 md

**职责：**
1. 更新用户画像.md — 补充新的生活细节、更新作息表、刷新睡眠问题诊断和趋势
2. 更新干预策略.md — 记录新的干预结果、调整当前策略、维护红线列表

**原则：**
- md 是给主 agent 看的，写成"一个了解用户的睡眠顾问会怎么给同事做交接"的语气
- 两份 md 合计控制在 ~800 token 以内
- 干预历史只保留最近 10 条，更早的提炼为规律后删除

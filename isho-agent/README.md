## 架构概览

```
用户打开 App
    │
    ▼
┌─────────────────────────────────────────────┐
│  07-events: 判断触发场景                      │
│  （首次打开？有待收集的反馈？用户主动发消息？）   │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  03-memory: 加载记忆                         │
│  （用户画像、建议历史、反馈历史）               │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  04-context-assembly: 拼装 prompt            │
│  （system + memory + tools + messages）      │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  05-agent-loop: 执行循环                     │
│  （模型推理 → 调用工具 → 观察结果 → 继续/停止）│
│  工具定义见 02-tools                          │
│  身份与原则见 01-system-prompt                │
│  输出规范见 06-output-style                   │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
              响应用户
```

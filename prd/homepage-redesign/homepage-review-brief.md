# 首页 PRD · 产品评审汇报一页纸

> 面向：产品评审 / 业务方
> 所属模块：首页
> 基线文档：`homepage-redesign-prd.md`、`today-tab-prd.md`

## 一、改版目标

Isho 以"精力"为中心。原首页把睡眠评分、心率、消耗、高能时段平铺成仪表盘，用户要自己拼出"今天精力好不好"。本次改版要把"精力"从一张普通卡片，重写为首屏 3 秒就能读懂的主角。

## 二、问题诊断（主 PRD §1.1 原文）

1. **缺少主线**：信息平铺无层级，用户需自行综合判断"今天精力好不好"
2. **核心指标弱化**：高能时长仅作普通卡片存在，未建立起用户心智
3. **缺乏行动指引**：看完数据不知道做什么，产品停留在"告诉你"
4. **场景未区分**：晨起和睡前两个场景用同一套布局

四条咬合在一起 —— 首页没有主角，也没有语气。

## 三、核心洞察（主 PRD §1.3 / §3.1）

1. **结论先行** → 核心判断由色彩 + 主指标直接承载，不依赖数字解读
2. **场景自适应** → 同一交互结构，内容随白天 / 夜间切换，"学一次即会"
3. **因果透明** → 归因与行动在同一屏说清，归因用偏离信号而非纯数值
4. **睡前安心不考核** → 夜间色彩、语言、推送全换一套语气

## 四、方案推导

- **合并状态 / 今日两 Tab 为"状态页 + 时间轴二级页"**：70% 用户只看第一层，仅 10% 深探时间分布（§3.1）；首屏答"现在怎么样"，点主指标下钻答"一天怎么走"。
- **状态页五层骨架**：主指标 + 短时变化 → 拟人化标题 → 3-4 张归因小卡 → 正文解读 → 3 条快捷提问 → 心率入口。一套骨架支撑全天四态。
- **四色自动切换（绿 / 橙 / 深灰 / 紫）**：场景自适应 → 色彩随节律切换，用户无需选择；同一骨架只换内容。
- **夜间 Hero 同区域变形、不另起一页**：布局对齐原则（§3.2.2）→ "入睡准备度"与"高能时长"共享 Hero，标签 / 数值 / 进度 / 色彩全程 crossfade。
- **主指标下方"较 2 小时前"**：日间关心近时动态而非隔日趋势，用更贴近当下的锚点。
- **归因 3 段区间罗盘**："罗盘不是进度条"（§3.2.3）→ 指针偏离方向即语义，比柱状图多一维信息。
- **时间轴二级页极简**：纵向能量曲线 + 小时刻度 + now 虚线，顶部回显高能时长与状态页一致。

## 五、评审验收锚点

- 状态页 3 秒内传达"当前怎么样"
- 四色切换对用户透明：同一骨架、同一交互，仅内容替换
- 睡前紫色调性为"安心不考核"，快捷提问用中性语言
- 状态页与今日时间轴的高能时长数据始终一致

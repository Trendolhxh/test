## 一、产品定位：状态页 + 今日时间轴

首页由两个页面协同组成，一页答"我怎么样"，一页答"我该做什么"。

| 页面 | 核心问题 | 产品角色 | 形态 |
|:---|:---|:---|:---|
| **状态页**（首页） | 我怎么样？ | 3 秒给结论的精力看板 | 默认落地页 |
| **今日时间轴**（二级页） | 我该做什么？ | 把事件投射到精力曲线上的匹配层 | 整屏跳转，返回键回状态页 |

**入口交互**：状态页的"高能时长"核心指标区域即为时间轴入口，点击整屏跳转进入今日时间轴。两页共享同一套底层数据，时间轴上的入睡窗口调整会反向同步回状态页。

## 二、用户使用场景

按一天的自然节律覆盖 5 个核心场景：

| 场景 | 触发时机 | 用户诉求 | 页面落点 |
|:---|:---|:---|:---|
| **晨起一眼看** | 当日首次打开 · 05:00–12:00 | 今天精力怎么样？重要事放什么时段？ | 状态页首屏 → 按需下探今日时间轴 |
| **日间查看** | 非首次打开 · 12:00 后 | 下一步该做什么？ | 今日时间轴（实时态） |
| **睡前常规** | 夜间模式激活后 | 现在适不适合睡？ | 状态页夜间模式（入睡准备度） |
| **重要日前夜** | 用户主动触发（Chatflow） | 明天要打仗，今晚怎么准备？ | 状态页 → Agent 生成建议 → 一键添加到时间轴 |
| **需要熬夜** | 用户调整入睡窗口 | 晚睡一小时代价多大？ | 时间轴入口 → 入睡窗口设置页 |

调性上，白天追求"清晰果断"，睡前追求"安心不考核"——不用红色、不用表情图标、不用"差/糟"字眼，临睡 30 分钟内不主动推送负面信息。

## 三、页面状态流转

### 3.1 状态页：白天 ↔ 夜间自动切换

核心指标区根据当日最后一个高能时段的结束时间自动切换，下层结构不变、仅内容替换：

```
白天模式                    夜间模式（最后一段高能结束后自动切换）
├ 今日高能时长 + 日变化   →  ├ "今日高能时长·已归零" 过渡条
                            └ 入睡准备度信号灯 + 推荐入睡时间 + 4 枚罗盘
下层：Smart Digest → 归因 → 异常 → 完整身体数据（两种模式共用此结构）
```

次日 05:00 后首次打开自动恢复白天模式。

### 3.2 页面跳转

```
状态页（首页，默认）
   │ 点击"高能时长"区域 / 夜间模式下推荐入睡时间区域
   ▼
今日时间轴（整屏新页，返回键回状态页）
   │ 点击时间轴上的 🌙 入睡窗口
   ▼
入睡窗口设置页（确认后回时间轴，两页数据同步刷新）
```

### 3.3 今日时间轴三态

按当前时间在 24h 时间轴上的位置自动切换，用户无需操作：

| 状态 | 触发 | 视觉 |
|:---|:---|:---|
| **预测态** | 晨起首次打开 | 全天虚线曲线 + 建议 pin 待执行，自动定位到起床时间 |
| **实时态** | 日间打开 | "现在"标记线，以上实线（已过）、以下虚线（未来） |
| **回顾态** | 睡前/夜间打开 | 全天实线，pin 标记完成/跳过，自动定位到入睡窗口 |

## 四、验收锚点

- 状态页首屏 3 秒内传达"今天怎么样"，不依赖文字解读
- 睡前场景不出现红色/徽章/"差评"文案，支持"隐藏分数"模式
- 状态页与今日时间轴的高能时长、入睡窗口等核心数据始终一致
- 时间轴三态切换对用户透明，无加载感知
